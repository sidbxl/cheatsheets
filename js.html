Modern Vanilla JS Cheatsheet

Focusing on modern ECMAScript (ES6+) features, browser APIs, and best practices. No legacy `var` or outdated techniques unless for comparison.

Basics & Linking JS

Including JavaScript in HTML

  • External File (Recommended): 
    
<script src="path/to/script.js" defer></script>

    defer: Downloads script with HTML, executes after HTML parsing (maintains order).
    async: Downloads script with HTML, executes as soon as available (may not maintain order, good for independent scripts).

  • Inline Script:

Variables & Data Types

Declaring Variables (ES6+)

  • let: Block-scoped variable, can be reassigned. 
    
let message = "Hello";
message = "World";
  • const: Block-scoped variable, must be initialized, cannot be reassigned. (Objects/arrays declared with const can still have their contents modified). 
    
const PI = 3.14;
// PI = 3.14159; // Error!

const person = { name: "Alice" };
person.name = "Bob"; // This is allowed

  • Avoid using var due to its function-scoping and hoisting quirks.

Primitive Data Types

  • String: Sequence of characters.
    let name = "Alice";
    Template Literals (ES6): let greeting = `Hello, ${name}!`;
  • Number: Integers and floating-point numbers.
    let age = 30; let price = 19.99;
  • Boolean: true or false.
    let isActive = true;
  • Null: Represents the intentional absence of any object value.
    let data = null;
  • Undefined: A variable that has been declared but not assigned a value.
    let score; // score is undefined
  • Symbol (ES6): Unique and immutable primitive value, often used as object keys.
    const id = Symbol('id');
  • BigInt (ES2020): For integers larger than Number.MAX_SAFE_INTEGER.
    const largeNumber = 1234567890123456789012345678901234567890n;

Non-Primitive Data Types (Objects)

  • Object: Collection of key-value pairs. 
    
let user = {
  firstName: "John",
  lastName: "Doe",
  age: 25
};
  • Array: Ordered list of values. 
    
let colors = ["red", "green", "blue"];
  • Function: A block of code designed to perform a particular task. (Functions are also objects).

Control Flow

  • if / else if / else 
    
if (condition1) {
  // block of code
} else if (condition2) {
  // block of code
} else {
  // block of code
}
  • switch 
    
switch(expression) {
  case value1:
    // code
    break;
  case value2:
    // code
    break;
  default:
    // code
}
  • Loops
    • for: for (let i = 0; i < 5; i++) { /* ... */ }
    • for...of (ES6, for iterables like Arrays, Strings): 
      
for (const item of myArray) {
  console.log(item);
}
    • for...in (for object properties - use with caution, iterates over prototype chain): 
      
for (const key in myObject) {
  if (myObject.hasOwnProperty(key)) {
    console.log(`${key}: ${myObject[key]}`);
  }
}
    • while: while (condition) { /* ... */ }
    • do...while: do { /* ... */ } while (condition);

Functions

  • Function Declaration: 
    
function greet(name) {
  return `Hello, ${name}!`;
}
  • Function Expression: 
    
const greet = function(name) {
  return `Hello, ${name}!`;
};
  • Arrow Functions (ES6): Concise syntax, lexical this binding. 
    
const add = (a, b) => a + b; // Implicit return for single expression

const multiply = (a, b) => {
  return a * b; // Explicit return for multi-line
};
  • Parameters:
    • Default Parameters (ES6): function sayHi(name = 'Guest') { /* ... */ }
    • Rest Parameters (ES6): Collects remaining arguments into an array. 
      
function sum(...numbers) {
  return numbers.reduce((acc, current) => acc + current, 0);
}
sum(1, 2, 3); // 6
  • Scope: Variables declared with let and const have block scope. Functions create their own scope.

Objects

  • Object Literals: 
    
const car = {
  make: 'Toyota',
  model: 'Camry',
  year: 2022,
  start: function() { // Method
    console.log('Engine started!');
  },
  honk() { // Shorthand method syntax (ES6)
    console.log('Beep beep!');
  }
};
car.start();
  • Accessing Properties: Dot notation (object.property) or bracket notation (object['property']).
  • this Keyword: Refers to the object the function is a method of (context depends on how the function is called). Arrow functions do not have their own this; they inherit it from the surrounding lexical scope.
  • Object Destructuring (ES6): 
    
const { make, model } = car;
console.log(make); // 'Toyota'

function printCarInfo({ make, model }) {
  console.log(`${make} ${model}`);
}
printCarInfo(car);
  • Property Shorthand (ES6): 
    
const name = 'Alice';
const age = 30;
const user = { name, age }; // Same as { name: name, age: age }

Arrays

  • Array Literals: const fruits = ['apple', 'banana', 'cherry'];
  • Accessing Elements: Zero-based index: fruits[0] // 'apple'
  • Common Properties & Methods:
    • .length: Number of elements.
    • .push(item): Adds to end. .pop(): Removes from end.
    • .unshift(item): Adds to start. .shift(): Removes from start.
    • .indexOf(item), .includes(item) (ES7)
    • .slice(start, end): Returns a new array (non-mutating).
    • .splice(start, deleteCount, ...itemsToAdd): Modifies original array.
  • Modern Iteration/Transformation Methods (ES6+): (These do not mutate the original array and return a new array or value) 
    
const numbers = [1, 2, 3, 4, 5];

numbers.forEach(num => console.log(num * 2));

const doubled = numbers.map(num => num * 2); // [2, 4, 6, 8, 10]

const evens = numbers.filter(num => num % 2 === 0); // [2, 4]

const sum = numbers.reduce((accumulator, currentValue) => accumulator + currentValue, 0); // 15

const firstEven = numbers.find(num => num % 2 === 0); // 2

const hasEven = numbers.some(num => num % 2 === 0); // true

const allPositive = numbers.every(num => num > 0); // true
  • Spread Syntax (ES6): For copying or merging arrays. 
    
const arr1 = [1, 2];
const arr2 = [...arr1, 3, 4]; // [1, 2, 3, 4]
const copy = [...arr1];
  • Array Destructuring (ES6): 
    
const [first, second, ...rest] = ['apple', 'banana', 'cherry', 'date'];
console.log(first); // 'apple'
console.log(rest); // ['cherry', 'date']

Events (Browser Environment)

Responding to user interactions or browser actions.


const button = document.querySelector('#myButton');

function handleClick(event) {
  console.log('Button clicked!', event);
  // event.target refers to the element that triggered the event
  // event.preventDefault(); // Can stop default browser action (e.g., form submission)
}

button.addEventListener('click', handleClick);

// To remove:
// button.removeEventListener('click', handleClick);

Common Event Types: click, mouseover, mouseout, mousedown, mouseup, mousemove, keydown, keyup, keypress, focus, blur, submit, change, load, DOMContentLoaded.

Event Delegation: Attach a single event listener to a parent element to manage events for multiple child elements. Check event.target inside the handler.

Asynchronous JavaScript

Handling operations that take time (e.g., network requests, timers) without blocking the main thread.

Callbacks (Traditional)

Functions passed as arguments to be executed later. Can lead to "callback hell" if nested deeply.

Promises (ES6)

An object representing the eventual completion (or failure) of an asynchronous operation.


const myPromise = new Promise((resolve, reject) => {
  // Asynchronous operation
  const success = true; // or false
  if (success) {
    resolve('Operation successful!');
  } else {
    reject('Operation failed.');
  }
});

myPromise
  .then(result => console.log(result)) // Handle success
  .catch(error => console.error(error)) // Handle failure
  .finally(() => console.log('Operation complete.')); // Runs regardless of success/failure (ES2018)

// Promise.all([promise1, promise2]), Promise.race([promise1, promise2])

Async/Await (ES2017)

Syntactic sugar over Promises, making asynchronous code look more synchronous.


async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error('Could not fetch data:', error);
  }
}
fetchData();

Fetch API (Browser Environment)

Modern interface for making network requests (replaces XMLHttpRequest).


// GET request
fetch('https://jsonplaceholder.typicode.com/todos/1')
  .then(response => {
    if (!response.ok) { throw new Error('Network response was not ok'); }
    return response.json(); // Parses JSON response
  })
  .then(data => console.log(data))
  .catch(error => console.error('Fetch error:', error));

// POST request
fetch('https://jsonplaceholder.typicode.com/posts', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    title: 'foo',
    body: 'bar',
    userId: 1
  })
})
.then(response => response.json())
.then(data => console.log(data));

Modules (ES6 Modules)

Organize code into reusable pieces. Must be served from a server (not file://) and used with <script type="module">.

math.js (Example Module):


export const PI = 3.14159;
export function add(a, b) {
  return a + b;
}
export default function multiply(a, b) { // Only one default export per module
  return a * b;
}

main.js (Using the Module):


import multiply, { PI, add as sum } from './math.js';
// 'multiply' is the default export, PI & add are named exports (add renamed to sum)

console.log(PI);
console.log(sum(2, 3));
console.log(multiply(4, 5));

Error Handling


try {
  // Code that might throw an error
  let result = riskyOperation();
  console.log(result);
} catch (error) {
  // Code to handle the error
  console.error('An error occurred:', error.message);
  // error.name, error.stack
} finally {
  // Code that runs regardless of an error (optional)
  console.log('Operation finished.');
}

// Throwing custom errors
function divide(a, b) {
  if (b === 0) {
    throw new Error('Division by zero is not allowed.');
  }
  return a / b;
}

Browser Storage (Browser Environment)

Storing data in the user's browser.

localStorage

Stores data with no expiration date. Data persists even after the browser window is closed.


// Set item
localStorage.setItem('username', 'Alice');

// Get item
const username = localStorage.getItem('username'); // 'Alice'

// Remove item
localStorage.removeItem('username');

// Clear all items
// localStorage.clear();

// Note: localStorage only stores strings. Use JSON.stringify/parse for objects.
const user = { name: 'Bob', id: 123 };
localStorage.setItem('user', JSON.stringify(user));
const retrievedUser = JSON.parse(localStorage.getItem('user'));

sessionStorage

Stores data for one session (data is lost when the browser tab is closed). API is the same as localStorage.


sessionStorage.setItem('sessionData', 'Temporary info');

JSON (JavaScript Object Notation)

A lightweight data-interchange format. Easy for humans to read/write, easy for machines to parse/generate.

  • JSON.stringify(object): Converts a JavaScript object or value to a JSON string. 
    
const data = { name: "Test", value: 42 };
const jsonString = JSON.stringify(data);
// jsonString is "{\"name\":\"Test\",\"value\":42}"
  • JSON.parse(jsonString): Parses a JSON string, constructing the JavaScript value or object described by the string. 
    
const jsonObject = JSON.parse(jsonString);
// jsonObject is { name: "Test", value: 42 }

Good Practices

  • Use Strict Mode: 'use strict'; at the beginning of your scripts or functions helps catch common coding errors. (Modules are strict by default).
  • Use let and const: Prefer const by default; use let if reassignment is necessary. Avoid var.
  • Write Clear, Readable Code: Use meaningful variable/function names, keep functions small and focused, comment complex logic.
  • Embrace Modern Features: Arrow functions, template literals, destructuring, spread/rest, Promises, async/await make code more concise and readable.
  • Handle Errors Gracefully: Use try...catch for synchronous errors and .catch() or try...catch with async/await for asynchronous errors.
  • Avoid Global Variables: Encapsulate code in functions or modules to prevent polluting the global scope.
  • Use Strict Equality: Prefer === and !== over == and != to avoid type coercion surprises.
  • Optimize DOM Manipulations: Minimize direct DOM access. Batch updates or use document fragments if creating many elements.
  • Use Event Delegation: For better performance when handling events on multiple child elements.
  • Keep Code DRY (Don't Repeat Yourself): Create reusable functions or modules for common tasks.
  • Code Linting & Formatting: Use tools like ESLint and Prettier to maintain code quality and consistency.

Bad Practices (Things to Avoid)

  • Using var: Leads to hoisting issues and less clear scoping.
  • Modifying Built-in Prototypes (e.g., Array.prototype.myMethod): Can lead to conflicts with future JS versions or other libraries.
  • Using eval(): Security risk and performance hit. There's almost always a better way.
  • Using document.write() after page load: Can overwrite the entire page.
  • Blocking the Main Thread: Long-running synchronous operations will freeze the UI. Use asynchronous patterns.
  • Creating "Callback Hell": Deeply nested callbacks are hard to read and maintain. Use Promises or async/await.
  • Ignoring Error Handling: Can lead to unexpected crashes or behavior.
  • Using innerHTML with Unsanitized User Input: Security risk (XSS attacks). Use textContent for text or sanitize HTML carefully.
  • Relying on Type Coercion Implicitly: Can lead to bugs. Be explicit about types or use strict comparisons.
  • Writing overly complex or "clever" code: Prioritize readability and maintainability.
