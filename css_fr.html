Aide-mémoire CSS Moderne

Centré sur les concepts CSS modernes, les bonnes pratiques et l'évitement des techniques obsolètes. Conçu pour compléter l'aide-mémoire HTML en utilisant le même fichier style.css.

Bases & Syntaxe

CSS (Cascading Style Sheets - Feuilles de Style en Cascade) décrit comment les éléments HTML doivent être affichés. Les règles se composent d'un sélecteur et d'une ou plusieurs déclarations.


sélecteur {
  propriété: valeur; /* Déclaration 1 */
  autre-propriété: autre-valeur; /* Déclaration 2 */
}
  • Sélecteur : Cible le ou les éléments HTML que vous souhaitez styler.
  • Déclaration : Contient une propriété et sa valeur.
  • Propriété : L'attribut de style que vous souhaitez modifier (par ex., color, font-size).
  • Valeur : La valeur de la propriété (par ex., blue, 16px).
  • Chaque déclaration se termine par un point-virgule (;).
  • Les déclarations sont contenues dans des accolades ({ }).

Lier CSS à HTML

Utilisez la balise <link> dans la section <head> :


<!DOCTYPE html>
<html lang="fr">
<head>
    
    <meta charset="UTF-8"> 
    <title>Titre de la Page</title> 
    <link rel="stylesheet" href="style.css">
</head>
<body>
    
</body>
</html>

Sélecteurs

Cibler les éléments est fondamental en CSS.

Sélecteur Exemple Description
Type p Sélectionne tous les éléments <p>.
Classe .ma-classe Sélectionne tous les éléments avec class="ma-classe".
ID #mon-id Sélectionne l'élément avec id="mon-id" (doit être unique).
Universel * Sélectionne tous les éléments.
Attribut [type="text"]
[data-abc]		 Sélectionne les éléments avec un attribut spécifique ou une paire attribut/valeur.
Descendant div p Sélectionne tous les éléments <p> à l'intérieur d'un <div>.
Enfant ul > li Sélectionne tous les éléments <li> qui sont des enfants directs d'un <ul>.
Frère Adjacent h2 + p Sélectionne le premier <p> suivant immédiatement un <h2>.
Frère Général h2 ~ p Sélectionne tous les éléments <p> qui sont frères d'un <h2> et viennent après lui.
Groupement h1, h2, p Sélectionne tous les éléments <h1>, <h2>, et <p>.

Pseudo-classes (:...) & Pseudo-éléments (::...)

Sélecteurs qui ciblent les éléments en fonction de leur état ou de leur position (pseudo-classes) ou stylent des parties spécifiques d'un élément (pseudo-éléments).

Type Exemple Description
Pseudo-classe a:hover Sélectionne un lien lorsque l'utilisateur le survole. Autres courants : :focus, :active, :visited, :first-child, :last-child, :nth-child(n), :not(sélecteur), :where(sélecteur), :is(sélecteur).
Pseudo-élément p::first-line Sélectionne la première ligne d'un <p>. Autres courants : ::before, ::after, ::marker, ::selection. (Note : un seul deux-points :before/:after fonctionne aussi pour des raisons historiques).

Unité Type Description

Le Modèle de Boîte (Box Model)

Chaque élément HTML est traité comme une boîte. Comprendre ses composants est crucial pour la mise en page.


+---------------------+
|       Marge         |
| +-----------------+ |
| |    Bordure      | |
| | +-------------+ | |
| | | Remplissage | | |
| | | +---------+ | | |
| | | | Contenu | | | |
| | | +---------+ | | |
| | |             | | |
| | +-------------+ | |
| |                 | |
| +-----------------+ |
|                     |
+---------------------+
  • Contenu : Le contenu réel (texte, image, etc.). Dimensionné par width/height.
  • Remplissage (Padding) : Espace entre le contenu et la bordure. Affecté par padding-top, padding-right, padding-bottom, padding-left, ou le raccourci padding.
  • Bordure (Border) : Une ligne entre le remplissage et la marge. Affectée par border-width, border-style, border-color, ou le raccourci border.
  • Marge (Margin) : Espace à l'extérieur de la bordure. Affecté par margin-top, margin-right, margin-bottom, margin-left, ou le raccourci margin. Les marges peuvent fusionner verticalement.

Propriété box-sizing

Contrôle la manière dont width et height sont calculées.

  • box-sizing: content-box; (Défaut) : width/height s'appliquent uniquement à la zone de contenu. Le remplissage et la bordure sont ajoutés *à l'extérieur* des dimensions spécifiées.
  • box-sizing: border-box; : width/height s'appliquent au contenu + remplissage + bordure. Ceci est souvent préféré car cela rend les calculs de mise en page plus intuitifs.

Pratique courante : Appliquer box-sizing: border-box; globalement ou à la plupart des éléments en utilisant un reset ou un style de base.


*, *::before, *::after {
  box-sizing: border-box;
}

Propriété Display

Contrôle le comportement de mise en page d'un élément.

  • display: block; : Commence sur une nouvelle ligne et occupe toute la largeur disponible (par ex., <p>, <div>, <h1>). Respecte largeur, hauteur, remplissage, marge.
  • display: inline; : Ne commence pas sur une nouvelle ligne et n'occupe que la largeur nécessaire (par ex., <span>, <a>, <strong>). Ignore largeur, hauteur, remplissage/marge haut/bas.
  • display: inline-block; : Combine les caractéristiques. S'affiche en ligne mais respecte largeur, hauteur, remplissage, marge (par ex., <img>, <input> par défaut).
  • display: none; : Masque complètement l'élément (le supprime du flux du document).
  • display: flex; : Fait de l'élément un conteneur flex, activant la mise en page Flexbox pour ses enfants directs.
  • display: grid; : Fait de l'élément un conteneur grid, activant la mise en page Grid pour ses enfants directs.

Mise en Page : Flexbox

Une méthode de mise en page unidimensionnelle pour arranger les éléments en lignes ou en colonnes.

Propriétés du Conteneur Flex (S'appliquent au parent avec display: flex;)

  • flex-direction: row | row-reverse | column | column-reverse; : Définit la direction des éléments.
  • justify-content: flex-start | flex-end | center | space-between | space-around | space-evenly; : Aligne les éléments le long de l'axe principal.
  • align-items: flex-start | flex-end | center | baseline | stretch; : Aligne les éléments le long de l'axe transversal.
  • flex-wrap: nowrap | wrap | wrap-reverse; : Contrôle le retour à la ligne si les éléments dépassent du conteneur.
  • gap: valeur; : Définit l'espacement entre les lignes et les colonnes (raccourci pour row-gap et column-gap).

Propriétés des Éléments Flex (S'appliquent aux enfants directs du conteneur flex)

  • flex-grow: nombre; : Capacité à s'agrandir si nécessaire (défaut 0).
  • flex-shrink: nombre; : Capacité à rétrécir si nécessaire (défaut 1).
  • flex-basis: valeur | auto; : Taille par défaut de l'élément avant la distribution de l'espace.
  • flex: grow shrink basis; : Raccourci pour flex-grow, flex-shrink, et flex-basis. Valeurs courantes : 1 1 auto, auto, none.
  • order: nombre; : Spécifie l'ordre d'un élément flexible par rapport aux autres (défaut 0).
  • align-self: auto | flex-start | flex-end | center | baseline | stretch; : Surcharge align-items du conteneur pour un seul élément.

Exemple : Centrer un div dans son parent


.conteneur {
  display: flex;
  justify-content: center; /* Centrer horizontalement */
  align-items: center; /* Centrer verticalement */
  min-height: 100vh; /* Hauteur d'exemple */
}

Mise en Page : Grid (Grille)

Une méthode de mise en page bidimensionnelle pour arranger les éléments en colonnes et en lignes.

Propriétés du Conteneur Grid (S'appliquent au parent avec display: grid;)

  • grid-template-columns: valeur(s); : Définit les colonnes de la grille (par ex., 1fr 1fr auto, repeat(3, 100px)).
  • grid-template-rows: valeur(s); : Définit les lignes de la grille.
  • gap: valeur; : Définit l'espacement entre les lignes et les colonnes (raccourci pour row-gap et column-gap).
  • justify-items: start | end | center | stretch; : Aligne les éléments dans leurs cellules le long de l'axe en ligne (ligne).
  • align-items: start | end | center | stretch; : Aligne les éléments dans leurs cellules le long de l'axe de bloc (colonne).
  • grid-template-areas: "zone1 zone2" "zone3 ."; : Définit les zones de la grille en les nommant dans une structure visuelle.

Propriétés des Éléments Grid (S'appliquent aux enfants directs du conteneur grid)

  • grid-column: début / fin; : Spécifie la taille et l'emplacement de la colonne (par ex., 1 / 3, span 2). Raccourci pour grid-column-start et grid-column-end.
  • grid-row: début / fin; : Spécifie la taille et l'emplacement de la ligne. Raccourci pour grid-row-start et grid-row-end.
  • grid-area: début-ligne / début-colonne / fin-ligne / fin-colonne; : Raccourci pour l'emplacement/dimensionnement, ou peut utiliser les noms de zones définis dans grid-template-areas.
  • justify-self: start | end | center | stretch; : Surcharge justify-items du conteneur pour un seul élément.
  • align-self: start | end | center | stretch; : Surcharge align-items du conteneur pour un seul élément.

Exemple : Une grille simple à 2 colonnes


.conteneur {
  display: grid;
  grid-template-columns: 1fr 2fr; /* Colonne gauche prend 1 part, droite prend 2 parts */
  gap: 20px; /* Espace entre les colonnes et les lignes */
}

Typographie

Mise en forme du contenu textuel.

  • font-family: "Nom de Police", générique; : Spécifie la police de caractères. Incluez toujours des polices génériques de secours (serif, sans-serif, monospace, cursive, fantasy).
  • font-size: valeur; : Spécifie la taille de la police (par ex., 16px, 1em, 1.5rem).
  • font-weight: normal | bold | 100 | ... | 900; : Spécifie le poids (graisse) de la police.
  • font-style: normal | italic | oblique; : Spécifie le style de la police.
  • line-height: valeur | nombre; : Définit la hauteur d'une boîte de ligne. Utilisez des nombres sans unité (par ex., 1.5) pour un meilleur héritage.
  • text-align: left | right | center | justify; : Aligne le texte dans son bloc conteneur.
  • text-decoration: none | underline | overline | line-through; : Ajoute des lignes de décoration au texte. Utilisez none pour supprimer les soulignements par défaut des liens.
  • color: valeur; : Définit la couleur du texte.

Chargement des Polices Web (par ex., Google Fonts, polices personnalisées)


@font-face {
  font-family: "Ma Police Perso";
  src: url("ma-police-perso.woff2") format("woff2"),
       url("ma-police-perso.woff") format("woff");
  font-weight: normal;
  font-style: normal;
}

body {
  font-family: "Ma Police Perso", sans-serif;
}

Couleur & Arrière-plan

Mise en forme des couleurs et des arrière-plans des éléments.

  • color: valeur; : Définit la couleur de premier plan (couleur du texte).
  • background-color: valeur; : Définit la couleur d'arrière-plan d'un élément.
  • background-image: url("chemin/vers/image.jpg") | linear-gradient(...); : Définit l'image d'arrière-plan.
  • background-repeat: repeat | no-repeat | repeat-x | repeat-y; : Comment l'image d'arrière-plan se répète.
  • background-position: valeur; : Positionne l'image d'arrière-plan (par ex., center top, 50% 50%).
  • background-size: auto | cover | contain | valeur; : Dimensionne l'image d'arrière-plan. cover remplit la zone, contain assure que toute l'image est visible.
  • background: raccourci; : Propriété raccourcie pour les styles d'arrière-plan. L'ordre compte : couleur image répétition position / taille attachement origine clip.

Valeurs de Couleur

  • #RRGGBB ou #RGB (Hexadécimal)
  • rgb(R, G, B) ou rgba(R, G, B, A) (Rouge, Vert, Bleu, Alpha/Opacité)
  • hsl(H, S, L) ou hsla(H, S, L, A) (Teinte, Saturation, Luminosité, Alpha)
  • Couleurs Nommées (par ex., red, blue, transparent)
  • Couleur Actuelle (currentColor) : Utilise la valeur calculée de color de l'élément.

Fonctionnalités CSS Moins Connues / Modernes

Tirer parti des nouvelles capacités pour un code plus propre et plus flexible.

  • Variables CSS (Propriétés Personnalisées) : Définir des valeurs réutilisables. 
    
:root { /* Définir globalement */
  --couleur-primaire: #1a73e8;
  --espacement-md: 16px;
}

h1 {
  color: var(--couleur-primaire);
}

.section {
  margin-bottom: var(--espacement-md);
}
  • Fonction calc() : Effectuer des calculs pour déterminer les valeurs CSS.
    Exemple : width: calc(100% - 40px);
  • Fonctions clamp(), min(), max() : Pour des valeurs responsives basées sur des contraintes.
    Exemple : font-size: clamp(1rem, 2.5vw, 2rem); (Maintient la taille de police entre 1rem et 2rem, s'adaptant fluidement en fonction de la largeur du viewport, dans la limite de 2.5vw).
  • object-fit & object-position : Contrôler comment le contenu d'un élément remplacé (comme <img> ou <video>) doit être redimensionné pour s'adapter à son conteneur.
    Exemple : object-fit: cover; (Similaire à background-size: cover).
  • Propriété filter : Appliquer des effets graphiques comme le flou, les niveaux de gris, le sépia, etc.
    Exemple : filter: grayscale(100%);
  • @layer (Couches de Cascade) : Organise les styles pour contrôler la cascade de manière plus explicite. (Utile pour les systèmes de design, CSS tiers). 
    
@layer reinitialisation, base, composants, utilitaires;

@layer base {
  body { margin: 0; }
}

@layer composants {
  .bouton { padding: 10px 20px; }
} /* La couche composants a une priorité plus élevée que la couche base */
  • Requêtes de Conteneur (@container) : Styler les éléments en fonction de la taille de leur *conteneur*, et non plus seulement du viewport. (Support encore relativement nouveau). 
    
.conteneur-carte {
  container-type: inline-size;
}

@container (min-width: 400px) {
  .carte {
    display: flex; /* La mise en page de la carte change si le conteneur est assez large */
  }
}

Concepts Clés : Spécificité, Cascade, Héritage

Comprendre ces règles est crucial pour prédire quels styles s'appliquent.

Spécificité

Un score attribué à un sélecteur CSS en fonction de ses composants. Une spécificité plus élevée l'emporte lorsque plusieurs règles ciblent le même élément et la même propriété.

  • Styles en ligne (la plus élevée)
  • ID (#mon-id) - 100 points
  • Classes (.ma-classe), Attributs ([type="text"]), Pseudo-classes (:hover) - 10 points
  • Éléments (p), Pseudo-éléments (::before) - 1 point
  • Sélecteur universel (*), Combinateurs (>, +, ~), Négation (:not()) - 0 point (mais le sélecteur interne compte)
  • !important (surcharge la spécificité - à utiliser avec précaution)

Exemple : #mon-id .ma-classe p (100 + 10 + 1 = 111 points) vs. .autre-classe p.surbrillance (10 + 1 + 10 = 21 points)

La Cascade

Lorsque plusieurs règles conflictuelles s'appliquent à un élément, le navigateur résout les conflits en fonction de (par ordre de priorité) :

  1. Origine : Styles de l'agent utilisateur < Styles utilisateur < Styles de l'auteur < !important de l'auteur < !important de l'utilisateur < !important de l'agent utilisateur (la plus élevée)
  2. Couche (Layer) : Les styles dans les instructions @layer ultérieures surchargent ceux des couches antérieures. Les styles non stratifiés ont une priorité plus élevée que les styles stratifiés.
  3. Spécificité : La règle avec la spécificité la plus élevée l'emporte.
  4. Ordre : Si la spécificité est égale, la dernière règle déclarée l'emporte.

Héritage

Certaines propriétés CSS (comme color, font-family, text-align) sont héritées par les éléments enfants si elles ne sont pas explicitement définies sur l'enfant. D'autres (comme border, margin, padding, width) ne le sont pas.

  • Utilisez le mot-clé inherit pour hériter explicitement d'une valeur de propriété du parent.
  • Utilisez initial pour définir une propriété à sa valeur par défaut.
  • Utilisez unset pour définir une propriété à sa valeur héritée si c'est une propriété héritée, et à sa valeur initiale sinon.
  • Utilisez revert pour réinitialiser une propriété à la valeur établie par la feuille de style de l'agent utilisateur (ou la feuille de style utilisateur le cas échéant).

Design Responsive (@media)

Adapter les mises en page et les styles en fonction des caractéristiques de la fenêtre d'affichage (viewport), généralement la largeur.


/* Approche Mobile-First : Styles par défaut pour petits écrans */
body {
  font-size: 1rem;
}

/* Styles pour les viewports plus larges que 768px */
@media (min-width: 768px) {
  body {
    font-size: 1.1rem;
  }
  .conteneur {
    width: 700px;
    margin: 0 auto;
  }
}

/* Styles pour les viewports plus larges que 1200px */
@media (min-width: 1200px) {
  .conteneur {
    width: 1100px;
  }
}
  • La règle @media applique des styles conditionnellement.
  • min-width : Applique les styles lorsque le viewport est *au moins* de la largeur spécifiée. (Courant pour l'approche mobile-first).
  • max-width : Applique les styles lorsque le viewport est *au plus* de la largeur spécifiée. (Courant pour l'approche desktop-first).
  • Autres fonctionnalités média : orientation, resolution, prefers-color-scheme (schéma de couleurs préféré), prefers-reduced-motion (mouvement réduit préféré), etc.

Bonnes Pratiques

  • Approche Mobile-First : Concevez et stylez d'abord pour les petits écrans, puis utilisez @media (min-width: ...) pour ajouter des styles pour les écrans plus grands. C'est souvent plus efficace.
  • Utilisez les Variables CSS : Pour des couleurs, espacements, polices, etc., cohérents. Rend les mises à jour beaucoup plus faciles.
  • Utilisez du HTML Sémantique : Les sélecteurs CSS fonctionnent mieux lorsqu'ils ciblent des éléments et des classes HTML significatifs, et non des classes nommées de manière présentationnelle (comme `.texte-rouge`).
  • Gardez les Sélecteurs Simples : Évitez l'imbrication profonde. Préférez les sélecteurs de classe aux combinateurs complexes ou aux sélecteurs d'ID pour le style lorsque cela est possible.
  • Utilisez rem pour les Tailles de Police : Permet aux utilisateurs de mettre à l'échelle le texte globalement dans les paramètres du navigateur. Utilisez em pour le remplissage/marge où vous voulez un espacement relatif à la taille de police de l'élément *actuel*.
  • Utilisez box-sizing: border-box; : Rend les calculs de largeur et de hauteur plus intuitifs.
  • Organisez et Commentez : Utilisez des commentaires (/* ... */) pour sectionner votre CSS. Organisez les règles logiquement (par ex., par composant, par section, en utilisant une méthodologie comme BEM, des classes utilitaires, etc.).
  • Optimisez les Performances : Minifiez le CSS, utilisez des sélecteurs efficaces, considérez le CSS critique pour le chargement initial, différez le CSS non critique.
  • Donnez la Priorité à l'Accessibilité : Assurez un contraste de couleurs suffisant, fournissez des états de focus pour les éléments interactifs (:focus), considérez prefers-reduced-motion pour les animations.
  • Utilisez Flexbox et Grid pour la Mise en Page : Ce sont les outils modernes et puissants pour organiser le contenu.
  • Validez Votre CSS : Utilisez des outils comme le Validateur CSS W3C.

Mauvaises Pratiques (À Éviter)

  • Utiliser Abusivement !important : Détruit la cascade, rend le débogage et les surcharges extrêmement difficiles. Devrait être utilisé avec parcimonie, voire pas du tout (par ex., dans les classes utilitaires ou pour surcharger des styles tiers).
  • Sélecteurs Trop Spécifiques ou Profondément Imbriqués : Mène à un code fragile, difficile à maintenir et à réutiliser. Augmente les problèmes de spécificité.
  • Utiliser les ID pour le Style : Les ID ont une spécificité élevée, ce qui les rend difficiles à surcharger. Ils devraient principalement être utilisés pour les accroches JavaScript ou les identificateurs de fragment dans les URL. Utilisez des classes pour le style.
  • Utiliser les Styles en Ligne (<div style="...">) : Difficile à gérer, à surcharger, et viole la séparation des préoccupations. Spécificité la plus élevée, difficile à supprimer ou à modifier ultérieurement.
  • Utiliser des Tableaux pour la Mise en Page : Les tableaux sont pour les données, pas pour l'arrangement visuel. Utilisez les méthodes de mise en page CSS.
  • Ignorer l'Accessibilité (par ex., faible contraste, pas de contours de focus) : Crée des expériences inutilisables pour de nombreux utilisateurs.
  • Utiliser des Valeurs en Pixels Fixes pour Tout : Rend le design responsive plus difficile. Utilisez des unités relatives comme rem, %, vw lorsque c'est approprié.
  • Coder en Dur des Couleurs/Valeurs de Manière Répétée : Rend les refontes et les mises à jour fastidieuses. Utilisez les Variables CSS.
  • Écrire du Code Dupliqué : Si vous vous retrouvez à écrire les mêmes paires propriété/valeur plusieurs fois, envisagez de regrouper les sélecteurs ou d'utiliser une classe/variable utilitaire.
  • Trop se Fier aux Valeurs par Défaut du Navigateur : Les navigateurs varient. Utilisez une réinitialisation CSS (reset) ou une feuille de style de normalisation (normalize) pour assurer la cohérence.
