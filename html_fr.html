Aide-mémoire HTML Moderne

Centré sur HTML en tant que standard vivant, le balisage sémantique et l'accessibilité. Aucune balise obsolète ou dépréciée.

Balises Minimales pour une Page HTML

Le minimum absolu requis pour un document HTML valide :


<!DOCTYPE html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Titre de la Page</title>
</head>
<body>
    
</body>
</html>
  • <!DOCTYPE html> : Déclare le type de document (essentiel).
  • <html lang="fr"> : L'élément racine. L'attribut lang est important pour l'accessibilité et les moteurs de recherche.
  • <head> : Contient des méta-informations sur le document HTML (non affichées).
    • <meta charset="UTF-8"> : Spécifie l'encodage des caractères (UTF-8 est fortement recommandé).
    • <meta name="viewport" content="width=device-width, initial-scale=1.0"> : Configure la fenêtre d'affichage (viewport) pour le design responsive.
    • <title>Titre de la Page</title> : Le titre qui apparaît dans l'onglet du navigateur ou la barre de titre de la fenêtre. Crucial pour le SEO et l'accessibilité.
  • <body> : Contient le contenu visible de la page.

Règles d'Échappement des Caractères (Entités HTML)

Pour afficher des caractères ayant une signification spéciale en HTML, ou des caractères difficiles à taper, utilisez les entités HTML :

Caractère Entité Nommée Entité Numérique Description
< &lt; &#60; Signe inférieur à
> &gt; &#62; Signe supérieur à
& &amp; &#38; Esperluette
" &quot; &#34; Guillemet double (Nécessaire dans les valeurs d'attributs entourées de guillemets doubles)
' &apos; &#39; Guillemet simple (Apostrophe) (Nécessaire dans les valeurs d'attributs entourées de guillemets simples)
  &nbsp; &#160; Espace insécable
© &copy; &#169; Symbole de copyright

Exemple : Pour afficher <p> comme texte, écrivez &lt;p&gt;.

Balises HTML Incontournables

Structure du Document & Sémantique

  • <header> : Contenu introductif pour une section ou une page (logos, navigation, recherche).
  • <nav> : Une section de liens de navigation.
  • <main> : Le contenu principal du document. Il ne devrait y avoir qu'un seul élément <main> par page qui ne soit pas caché.
  • <article> : Contenu autonome, distribuable indépendamment (par ex., article de blog, nouvelle).
  • <section> : Un regroupement thématique de contenu, typiquement avec un titre.
  • <aside> : Contenu tangentiellement lié au contenu qui l'entoure (par ex., barre latérale, citations).
  • <footer> : Pied de page pour une section ou une page (copyright, liens connexes).
  • <h1> à <h6> : Niveaux de titre. Utilisez-les hiérarchiquement.
  • <p> : Un paragraphe de texte.
  • <div> : Un conteneur générique pour le contenu de flux. À utiliser lorsqu'aucun autre élément sémantique n'est approprié (souvent pour des raisons de style/mise en page avec CSS).
  • <span> : Un conteneur générique en ligne pour le contenu phrasé. À utiliser pour styliser des parties de texte ou lorsqu'aucun autre élément sémantique en ligne n'est approprié.

Contenu Textuel

  • <a href="url">Texte du Lien</a> : Un hyperlien.
    • target="_blank" : Ouvre le lien dans un nouvel onglet/fenêtre. Ajoutez rel="noopener noreferrer" pour la sécurité.
    • download : Invite l'utilisateur à télécharger la ressource liée.
  • <strong> : Indique un texte d'une grande importance, gravité ou urgence. (Typiquement en gras).
  • <em> : Indique une emphase accentuée. (Typiquement en italique).
  • <b> : Texte décalé stylistiquement sans transmettre d'importance supplémentaire (par ex., mots-clés). Utilisez CSS pour le style si possible.
  • <i> : Texte dans une voix ou une humeur alternative (par ex., terme technique, phrase étrangère). Utilisez CSS pour le style si possible.
  • <br> : Un saut de ligne. À utiliser avec parcimonie, uniquement pour les sauts de ligne réels dans le texte (comme les poèmes ou les adresses).
  • <hr> : Une rupture thématique (par ex., changement de scène dans une histoire, changement de sujet). Souvent affiché comme une règle horizontale.
  • <blockquote> : Représente une section citée d'une autre source. Utilisez l'attribut cite pour l'URL de la source.
  • <q> : Une citation en ligne. Les navigateurs ajoutent souvent des guillemets. Utilisez l'attribut cite pour l'URL de la source.
  • <cite> : Représente une référence à une œuvre créative (par ex., titre de livre, chanson, film).
  • <sub> & <sup> : Texte en indice et en exposant.
    Exemple : H2O, E = mc2
  • <ins> & <del> : Représente du texte inséré ou supprimé, souvent pour des modifications.
    Exemple : Ce texte a été supprimé ajouté.

Listes

  • <ul> : Une liste non ordonnée (à puces).
  • <ol start="nombre" reversed type="a/A/i/I/1"> : Une liste ordonnée (numérotée).
    • start : Numéro de départ.
    • reversed : Inverse la numérotation.
    • type : Style de numérotation (a, A, i, I, 1).
  • <li> : Un élément de liste (utilisé dans <ul>, <ol>, <menu>).
  • <dl>, <dt>, <dd> : Liste de descriptions, terme et description. 
    
<dl>
  <dt>HTML</dt>
  <dd>Langage de Balisage HyperTexte</dd>
</dl>

Images & Médias

  • <img src="chemin/vers/image.jpg" alt="Texte descriptif" width="X" height="Y"> : Intègre une image.
    • src : Chemin vers l'image.
    • alt : Essentiel pour l'accessibilité (texte alternatif si l'image ne se charge pas ou pour les lecteurs d'écran). Laissez vide (alt="") pour les images purement décoratives.
    • width & height : Spécifiez les dimensions pour éviter les décalages de mise en page. Peut être surchargé par CSS. Utilisez les dimensions intrinsèques si possible.
    • srcset & sizes : Pour les images responsives (fournir plusieurs sources d'images pour différentes résolutions/tailles de viewport).
  • <audio controls src="audio.mp3">Texte de remplacement</audio> : Intègre du contenu audio.
    Peut utiliser <source src="audio.ogg" type="audio/ogg"> pour plusieurs formats.
  • <video controls width="X" height="Y" src="video.mp4" poster="image.jpg">Texte de remplacement</video> : Intègre du contenu vidéo.
    Peut utiliser <source src="video.webm" type="video/webm"> pour plusieurs formats.
  • <iframe src="url" title="Titre descriptif"></iframe> : Intègre un autre document HTML. L'attribut title est important pour l'accessibilité.

Formulaires

Crucial pour l'interaction utilisateur. Associez toujours les étiquettes aux contrôles de formulaire.


<form action="/url-de-soumission" method="post">
    <label for="nomutilisateur">Nom d'utilisateur :</label>
    <input type="text" id="nomutilisateur" name="nomutilisateur" required>

    <input type="submit" value="Soumettre">
</form>
  • <form action="url" method="get|post"> : Définit un formulaire HTML.
  • <label for="idElement">Texte de l'étiquette</label> : Associe une étiquette à un contrôle de formulaire. Cliquer sur l'étiquette active/donne le focus au contrôle.
  • <input type="..."> : L'élément de formulaire le plus polyvalent.
    • Attributs type courants : text, password, checkbox, radio, submit, button, file, date, email, number, tel, url, search, color, range, hidden.
    • Attributs courants : id, name, value, placeholder, required, disabled, readonly, checked (pour radio/checkbox), min, max, step (pour number/range).
  • <textarea name="message" rows="4" cols="50"></textarea> : Une zone de saisie de texte multiligne.
  • <select name="options"> & <option value="val">Texte</option> : Une liste déroulante.
    • <optgroup label="Nom du Groupe"> : Regroupe les options dans un <select>.
  • <button type="submit|reset|button">Texte du Bouton</button> : Un bouton cliquable. Peut contenir du contenu HTML, contrairement à <input type="button">.
  • <fieldset> & <legend> : Regroupe les contrôles associés dans un formulaire, avec une légende. 
    
<fieldset>
  <legend>Coordonnées</legend>
  
</fieldset>
  • <datalist id="items"> & <option value="Choix"> : Fournit des options prédéfinies pour un élément <input> (autocomplétion). Liez avec <input list="items">.

Tableaux

Pour les données tabulaires, pas pour la mise en page.


<table id="monthly-sales-example-table">
    <caption>Ventes Mensuelles</caption>
    <thead>
        <tr>
            <th scope="col">Mois</th>
            <th scope="col">Ventes</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Janvier</td>
            <td>1000 €</td>
        </tr>
    </tbody>
    <tfoot>
        <tr>
            <td>Total</td>
            <td>1000 €</td>
        </tr>
    </tfoot>
</table>
  • <table> : Le conteneur principal du tableau.
  • <caption> : Titre/légende du tableau. Important pour l'accessibilité.
  • <thead> : Regroupe le contenu de l'en-tête dans un tableau.
  • <tbody> : Regroupe le contenu du corps dans un tableau.
  • <tfoot> : Regroupe le contenu du pied de page dans un tableau.
  • <tr> : Une ligne de tableau.
  • <th scope="col|row"> : Une cellule d'en-tête de tableau. L'attribut scope améliore l'accessibilité.
  • <td> : Une cellule de données de tableau.
  • Attributs colspan / rowspan sur <th> ou <td> : Font qu'une cellule s'étend sur plusieurs colonnes/lignes.

Bonnes Pratiques

  • HTML Sémantique : Utilisez les balises HTML pour leur usage prévu. Cela améliore l'accessibilité, le SEO et la maintenabilité. (par ex., utilisez <button> pour les boutons, pas un <div> stylisé).
  • Accessibilité (a11y) :
    • Utilisez toujours du texte alt pour les images (alt="" pour les images décoratives).
    • Assurez un contraste de couleurs suffisant.
    • Utilisez les attributs ARIA (Accessible Rich Internet Applications) lorsque nécessaire pour fournir une sémantique supplémentaire aux technologies d'assistance, mais donnez la priorité à la sémantique HTML native.
    • Assurez la navigabilité au clavier.
    • Utilisez <label> pour les contrôles de formulaire.
  • Validez Votre HTML : Utilisez des outils comme le Validateur W3C pour détecter les erreurs.
  • Séparez les Préoccupations :
    • HTML pour la structure.
    • CSS pour la présentation/le style.
    • JavaScript pour le comportement/l'interactivité.
    Évitez les styles en ligne (<div style="...">) et les gestionnaires d'événements de script en ligne (<button onclick="...">) autant que possible.
  • Indentation Correcte & Lisibilité : Gardez votre code propre et bien formaté.
  • Commentaires : Utilisez des commentaires (<!-- ... -->) pour expliquer les parties complexes de votre balisage.
  • Utilisez les Titres Hiérarchiquement : Ne sautez pas de niveaux de titre (par ex., <h1> puis <h3>). Structurez votre contenu logiquement.
  • Encodage des Caractères : Déclarez toujours <meta charset="UTF-8">.
  • Déclaration de Langue : Définissez la langue de votre page avec <html lang="fr">.
  • Design Responsive : Incluez <meta name="viewport" ...> et utilisez des techniques responsives (requêtes média CSS, grilles flexibles).
  • Fermez les Balises : Bien que certaines balises soient auto-fermantes (par ex. <img>, <br>, <input>) ou optionnelles dans certains contextes, il est généralement bon de fermer tous les éléments non vides (par ex. <p>...</p>, <div>...</div>). Dans la syntaxe XHTML (comme en JSX), les balises auto-fermantes nécessitent une barre oblique finale (<img />). La syntaxe HTML pour les éléments vides ne l'exige pas.

Mauvaises Pratiques (À Éviter)

  • Utiliser des Tableaux pour la Mise en Page : Les tableaux sont pour les données tabulaires. Utilisez CSS (Flexbox, Grid) pour la mise en page.
  • Utiliser Abusivement les Styles en Ligne : <p style="color: red;"> rend la maintenance difficile. Utilisez des fichiers CSS externes.
  • Ne Pas Utiliser de Texte alt pour les Images : C'est mauvais pour l'accessibilité et le SEO.
  • Utiliser des Balises/Attributs Dépréciés : Par ex., <font>, <center>, <bgcolor>, <frame>, <frameset>, <marquee>. Ils ne font plus partie du standard HTML.
  • "Divitis" ou "Spanitis" : Surutiliser <div> et <span> lorsque des balises plus sémantiques sont appropriées.
  • Sauter des Niveaux de Titre : Par ex., passer d'un <h1> directement à un <h3>. Cela casse la structure du document et perturbe les technologies d'assistance.
  • Utiliser <br> pour l'Espacement : Utilisez les marges/paddings CSS pour l'espacement de la mise en page.
  • Ne Pas Fermer Correctement les Balises : Bien que les navigateurs soient indulgents, cela peut entraîner des problèmes de rendu inattendus et est invalide pour certains éléments.
  • Mettre des Éléments de Type Bloc à l'Intérieur d'Éléments en Ligne : Par exemple, <a href="#"><div>...</div></a> est généralement incorrect (bien que les règles d'analyse HTML5 soient plus indulgentes pour <a>, il est toujours préférable de l'éviter si un <div> peut simplement être stylisé comme un lien, ou si le lien englobe du contenu en ligne). La règle est : le contenu phrasé (en ligne) peut aller à l'intérieur du contenu de flux (bloc), mais généralement pas l'inverse, sauf pour des éléments spécifiques comme <a> qui peut englober du contenu de flux.
  • Écrire du Texte de Lien Générique : "Cliquez ici" ou "En savoir plus" est mauvais pour l'accessibilité. Le texte du lien doit être descriptif de la destination du lien.
