Modern WordPress Cheatsheet

A quick reference for WordPress developers and advanced users, focusing on core concepts, theme/plugin development basics, and common APIs.

1. Introduction to WordPress

WordPress is a popular open-source Content Management System (CMS) built on PHP and MySQL. It powers a significant portion of the web.

  • Self-Hosted WordPress (WordPress.org): The free software you download and install on your own web hosting. Offers full control and flexibility.
  • WordPress.com: A commercial hosting service that uses a modified version of WordPress. Simpler to start, but with limitations on customization depending on the plan. This cheatsheet focuses on self-hosted WordPress.
  • Key Concepts: Posts, Pages, Themes, Plugins, Widgets, Menus, The Loop.

The Loop

The Loop is the PHP code used by WordPress to display posts. Most theme template files will use some variation of The Loop.


<?php if ( have_posts() ) : ?>
    <?php while ( have_posts() ) : the_post(); ?>
        // Display post content using template tags like the_title(), the_content(), etc.
        <h2><a href="<?php the_permalink(); ?>"><?php the_title(); ?></a></h2>
        <div class="entry-content">
            <?php the_content(); ?>
        </div>
    <?php endwhile; ?>
    // Add pagination if needed
<?php else : ?>
    <p><?php _e( 'Sorry, no posts matched your criteria.' ); ?></p>
<?php endif; ?>

2. Installation & Basic Configuration

Setting up a self-hosted WordPress site involves a few key steps.

  • Prerequisites: Web server (Apache/Nginx), PHP (check official requirements), MySQL or MariaDB database.
  • Download WordPress from WordPress.org.

wp-config.php

This is one of the most important files, typically created during installation by copying wp-config-sample.php and filling in your database details. It also contains unique Authentication Keys and Salts which are automatically generated during setup; these enhance security and users typically do not need to modify them manually unless specifically advised (e.g., after a security incident).

Key User-Modifiable Constants:

ConstantDescription
DB_NAMEYour database name.
DB_USERYour database username.
DB_PASSWORDYour database password.
DB_HOSTDatabase host (usually 'localhost').
$table_prefixDatabase table prefix (e.g., 'wp_'). Change for security on new installs.
WP_DEBUGSet to true for development to show errors. Default false.
WP_DEBUG_LOGSet to true to log errors to wp-content/debug.log (if WP_DEBUG is true).
WP_DEBUG_DISPLAYSet to false to hide errors on screen (if WP_DEBUG is true).
WP_MEMORY_LIMITCan increase PHP memory limit for WordPress (e.g., '128M').

Admin Settings

After installation, review these settings in the WordPress admin dashboard:

  • Settings > General: Site Title, Tagline, WordPress Address (URL), Site Address (URL), Email Address, Timezone.
  • Settings > Reading: Homepage displays (your latest posts or a static page).
  • Settings > Permalinks: Choose a URL structure (e.g., "Post name" is common for SEO). Save to generate/update .htaccess.

3. The Admin Dashboard

The backend interface for managing your WordPress site (usually yourdomain.com/wp-admin).

  • Dashboard: Overview, quick links.
  • Posts: Manage blog articles. Categories and Tags for organization.
  • Media: Upload and manage images, videos, and other files.
  • Pages: Manage static content like "About Us" or "Contact."
  • Comments: Moderate user comments.
  • Appearance:
    • Themes: Install, activate, and customize themes.
    • Customize: Live theme customizer (options vary by theme).
    • Widgets: Add content blocks to sidebars, footers, etc.
    • Menus: Create and manage navigation menus.
    • Theme File Editor: (Use with extreme caution, direct theme file editing).
  • Plugins: Install, activate, and manage plugins to extend functionality.
  • Users: Manage user accounts and roles (Administrator, Editor, Author, Contributor, Subscriber).
  • Tools: Import/Export data, Site Health.
  • Settings: Configure various aspects of your site (General, Writing, Reading, Discussion, Media, Permalinks, Privacy).

Block Editor (Gutenberg)

The default content editor for posts and pages. Uses "blocks" for different content types (paragraphs, headings, images, lists, columns, etc.).

4. Theme Development Basics

Themes control the visual presentation of your WordPress site.

Minimal Theme Structure (in wp-content/themes/your-theme-name/)

  • style.css: Contains theme header information (Name, Author, Version, etc.) and CSS styles. 
    
/*
Theme Name: My Awesome Theme
Theme URI: https://example.com/my-awesome-theme
Author: Your Name
Author URI: https://example.com
Description: A brief description of the theme.
Version: 1.0
License: GNU General Public License v2 or later
License URI: http://www.gnu.org/licenses/gpl-2.0.html
Text Domain: my-awesome-theme
Tags: one-column, custom-background, custom-logo, editor-style
*/
  • index.php: The main fallback template file if more specific ones aren't found. Usually contains The Loop.
  • functions.php: Acts like a plugin for your theme. Used to add features, enqueue scripts/styles, register menus/sidebars, etc.
  • screenshot.png: A 1200x900px image representing your theme in the Appearance > Themes admin screen.

Common Template Files & Template Hierarchy

WordPress uses a template hierarchy to decide which file to use for displaying different types of content.

  • header.php: Site header (<head>, opening <body>, site title, main navigation). Loaded with <?php get_header(); ?>.
  • footer.php: Site footer (closing tags, scripts). Loaded with <?php get_footer(); ?>.
  • sidebar.php: Sidebar content. Loaded with <?php get_sidebar(); ?>.
  • front-page.php: Used for the site's front page (if set to "Static Page" in Reading Settings and this file exists).
  • home.php: Used for the blog posts index (if front page is static, or if "Your latest posts" is selected and this file exists).
  • single.php: Displays a single blog post (post post type).
  • page.php: Displays a single Page (page post type).
  • archive.php: Displays archives (category, tag, author, date). More specific versions: category.php, tag.php, author.php, date.php.
  • search.php: Displays search results.
  • 404.php: Displays when content is not found.
  • comments.php: Displays comments and comment form.

Essential Theme Functions (Template Tags)

  • <?php bloginfo('name'); ?>: Displays site title.
  • <?php the_title(); ?>: Displays the post/page title (inside The Loop).
  • <?php the_content(); ?>: Displays post/page content (inside The Loop).
  • <?php the_excerpt(); ?>: Displays post excerpt (inside The Loop).
  • <?php the_permalink(); ?>: Displays the URL of the current post/page (inside The Loop).
  • <?php the_post_thumbnail(); ?>: Displays the featured image (inside The Loop). Needs theme support: <?php add_theme_support('post-thumbnails'); ?> in functions.php.
  • <?php get_template_part('template-parts/content', 'page'); ?>: Loads a template part (e.g., content-page.php from template-parts folder).
  • <?php wp_head(); ?>: Hook. Place just before </head> in header.php. Essential for plugins.
  • <?php wp_footer(); ?>: Hook. Place just before </body> in footer.php. Essential for plugins.
  • <?php body_class(); ?>: Adds useful CSS classes to the <body> tag.

Enqueueing Styles & Scripts (in functions.php)


<?php
function mytheme_enqueue_scripts() {
    // Enqueue main stylesheet
    wp_enqueue_style( 'mytheme-style', get_stylesheet_uri() ); // For parent theme style.css
    // For child theme, parent style: wp_enqueue_style( 'parent-style', get_template_directory_uri() . '/style.css' );

    // Enqueue a custom CSS file
    wp_enqueue_style( 'mytheme-custom', get_template_directory_uri() . '/css/custom.css', array(), '1.0.0' );

    ) );
}
add_action( 'wp_enqueue_scripts', 'mytheme_enqueue_scripts' );
?>

5. Plugin Development Basics

Plugins extend WordPress functionality without modifying core files or themes.

Minimal Plugin Structure (in wp-content/plugins/your-plugin-name/your-plugin-file.php)

A single PHP file with a standard plugin header comment:


<?php
/**
 * Plugin Name:       My Awesome Plugin
 * Plugin URI:        https://example.com/my-awesome-plugin
 * Description:       A brief description of what the plugin does.
 * Version:           1.0.0
 * Author:            Your Name
 * Author URI:        https://example.com
 * License:           GPL v2 or later
 * License URI:       https://www.gnu.org/licenses/gpl-2.0.html
 * Text Domain:       my-awesome-plugin
 * Domain Path:       /languages
 */

// Prevent direct access to the file
if ( !defined( 'ABSPATH' ) ) {
    exit; // Exit if accessed directly
}

// Your plugin code goes here...
function my_awesome_plugin_activation() {
    // Code to run on plugin activation
}
register_activation_hook( __FILE__, 'my_awesome_plugin_activation' );

function my_awesome_plugin_deactivation() {
    // Code to run on plugin deactivation
}
register_deactivation_hook( __FILE__, 'my_awesome_plugin_deactivation' );
?>

Hooks: Actions & Filters

Hooks are the foundation of plugin (and theme) development, allowing you to "hook into" WordPress core or other plugins/themes.

  • Actions (add_action, do_action): Execute a function at a specific point in WordPress execution. They don't return anything. 
    
<?php
// Correct action example: Add something to the footer
function myplugin_custom_footer_text() {
    echo '
    My Plugin Footer Text
    '    ;
}
add_action( 'wp_footer', 'myplugin_custom_footer_text', 99 ); // 99 is priority
?>
  • Filters (add_filter, apply_filters): Modify data before it's used or displayed. They *must* return a value. 
    
<?php
// Example: Modify the post title
function myplugin_modify_title( $title ) {
    if ( in_the_loop() && is_main_query() ) {
        return 'Prefix: ' . $title;
    }
    return $title;
}
add_filter( 'the_title', 'myplugin_modify_title' );

// Example: Filter 'the_content'
function myplugin_add_to_content( $content ) {
    if ( is_single() && in_the_loop() && is_main_query() ) {
        $content .= '
    Thanks for reading! (Via Filter)
    '    ;
    }
    return $content;
}
add_filter( 'the_content', 'myplugin_add_to_content' );
?>

Creating Shortcodes (in functions.php or plugin)

Allow users to embed dynamic content in posts/pages using a simple tag like [my_shortcode].


<?php
function my_custom_shortcode_function( $atts, $content = null ) {
    // $atts are attributes passed to the shortcode, e.g., [my_shortcode foo="bar"]
    // $content is the content between opening and closing tags, e.g., [my_shortcode]Content here[/my_shortcode]
    $a = shortcode_atts( array(
        'message' => 'Hello World!',
    ), $atts );

    return '
' . esc_html( $a['message'] ) . esc_html( $content ) . '
';
}
add_shortcode( 'my_shortcode', 'my_custom_shortcode_function' );
?>

Plugin Best Practices

  • Prefix Everything: Functions, classes, constants, global variables, hooks, CSS classes to avoid conflicts (e.g., myplugin_function_name).
  • Security: Sanitize inputs, escape outputs, use nonces.
  • Check for Existing Functions/Classes: <?php if ( !function_exists('some_function') ) { ... } ?>
  • Documentation: Use PHPDoc blocks for functions and classes.

6. Common WordPress Functions & APIs

WP_Query

For creating custom loops and fetching posts based on specific criteria. More powerful than simple get_posts().


<?php
$args = array(
    'post_type'      => 'post',
    'posts_per_page' => 5,
    'category_name'  => 'news',
    'orderby'        => 'date',
    'order'          => 'DESC',
    // Many more arguments available...
);
$custom_query = new WP_Query( $args );

if ( $custom_query->have_posts() ) :
    while ( $custom_query->have_posts() ) : $custom_query->the_post();
        // Display post content
        the_title();
    endwhile;
    wp_reset_postdata(); // IMPORTANT: Restore original Post Data
else :
    // No posts found
endif;
?>

Options API

  • <?php get_option( 'option_name', $default_value ); ?>: Retrieves an option value from the wp_options table.
  • <?php update_option( 'option_name', $new_value ); ?>: Updates an option value. Creates it if it doesn't exist.
  • <?php add_option( 'option_name', $value, '', 'yes' ); ?>: Adds an option if it doesn't exist.
  • <?php delete_option( 'option_name' ); ?>: Deletes an option.

Metadata API (Post Meta, User Meta, Term Meta, Comment Meta)

For storing custom data associated with posts, users, terms, or comments.


<?php
// For Post Meta (similar functions exist for users, terms, comments)
$post_id = get_the_ID(); // Example: inside The Loop

// Add
add_post_meta( $post_id, 'my_custom_field', 'Field Value', true ); // true for unique key

// Get
$meta_value = get_post_meta( $post_id, 'my_custom_field', true ); // true for single value

// Update
update_post_meta( $post_id, 'my_custom_field', 'New Value' );

// Delete
delete_post_meta( $post_id, 'my_custom_field' );
?>

Taxonomy Functions

  • <?php get_the_terms( $post_id, 'taxonomy_slug' ); ?>: Get terms for a post in a specific taxonomy.
  • <?php wp_get_post_terms( $post_id, 'taxonomy_slug', $args ); ?>: More flexible term retrieval.
  • <?php get_categories( $args ); ?>, <?php get_tags( $args ); ?>
  • <?php register_taxonomy( 'custom_taxonomy', 'post_type', $args ); ?>: Create custom taxonomies.
  • <?php register_post_type( 'custom_post_type', $args ); ?>: Create custom post types.

User Functions

  • <?php get_users( $args ); ?>: Retrieve a list of users.
  • <?php get_current_user_id(); ?>: Get ID of the currently logged-in user. Returns 0 if not logged in.
  • <?php wp_get_current_user(); ?>: Get WP_User object for current user.
  • <?php current_user_can( 'capability' ); ?>: Check if current user has a specific capability (e.g., 'edit_posts').
  • <?php is_user_logged_in(); ?>: Checks if a user is logged in.

Internationalization (i18n) & Localization (L10n)

Making your theme/plugin translatable.

  • <?php __( 'Text to translate', 'text-domain' ); ?>: Retrieve translated string.
  • <?php _e( 'Text to translate', 'text-domain' ); ?>: Echo translated string.
  • <?php _n( 'Singular', 'Plural', $number, 'text-domain' ); ?>: Retrieve translated string for pluralization.
  • <?php esc_html__(); ?>, <?php esc_attr_e(); ?>, etc.: Versions that also escape the output for security.
  • Load text domain in plugin/theme: <?php load_plugin_textdomain(); ?> or <?php load_theme_textdomain(); ?>.

7. Security & Maintenance Best Practices

  • Keep Everything Updated: WordPress core, themes, and plugins.
  • Use Strong, Unique Passwords: For admin accounts, database, FTP. Consider a password manager.
  • Limit User Privileges: Assign users the lowest role necessary for their tasks.
  • Regular Backups: Database and files. Many hosting providers offer this, or use backup plugins.
  • Sanitize Inputs: Clean any data coming from users or external sources before using it (e.g., <?php sanitize_text_field(); ?>, <?php absint(); ?>).
  • Escape Outputs: Securely display data to prevent XSS attacks (e.g., <?php esc_html(); ?>, <?php esc_attr(); ?>, <?php esc_url(); ?>, <?php wp_kses_post(); ?>).
  • Use Nonces (Numbers used once): Verify intent and prevent CSRF attacks for actions and form submissions.
    • <?php wp_create_nonce('my_action_nonce'); ?>: Generate a nonce.
    • <?php wp_verify_nonce($_POST['my_nonce_field'], 'my_action_nonce'); ?>: Verify a nonce.
    • <?php wp_nonce_field('my_action_nonce', 'my_nonce_field_name'); ?>: Output hidden nonce field in a form.
  • Choose Reputable Themes & Plugins: From trusted sources, well-maintained.
  • Consider a Security Plugin: (e.g., Wordfence, Sucuri) for extra protection, but don't rely on them solely.
  • Disable File Editing: Add <?php define('DISALLOW_FILE_EDIT', true); ?> to wp-config.php.

8. Introduction to Block Themes & Full Site Editing (FSE)

Full Site Editing (FSE) represents a significant evolution in WordPress, allowing users to design their entire site—from header to footer—using blocks via the Site Editor. Block Themes are themes built specifically to leverage FSE capabilities.

  • What it is: FSE aims to provide a unified block-based experience for all parts of a site, not just post/page content. This includes headers, footers, sidebars, and page templates.
  • Key Differences from Classic Themes:
    • theme.json: A central configuration file that controls global styles, block styles, settings (like color palettes, typography presets), and layout options. It replaces many traditional add_theme_support() calls and customizer settings. 
      
// Very simplified theme.json example snippet
{
  "version": 2,
  "settings": {
    "color": {
      "palette": [
        { "name": "Primary", "slug": "primary", "color": "#0073aa" }
      ]
    },
    "typography": {
      "fontSizes": [
        { "name": "Small", "slug": "small", "size": "1rem" }
      ]
    }
  },
  "styles": { /* Global styles for elements/blocks */ },
  "templateParts": [ /* Define template parts */ ],
  "customTemplates": [ /* Define custom page templates */ ]
}
    • Block-Based Templates: Templates (like index.html, single.html in a templates folder) and Template Parts (like header.html, footer.html in a parts folder) are primarily composed of HTML comments representing blocks, rather than extensive PHP logic. 
      
<!-- wp:group {"layout":{"type":"constrained"}} -->
<div class="wp-block-group">
    <!-- wp:site-title /-->
    <!-- wp:navigation /-->
</div>
<!-- /wp:group -->
    • Site Editor: (Appearance > Editor) A visual interface to modify templates, template parts, and global styles directly using blocks.
    • Reduced PHP in Templates: While PHP can still be used (e.g., for dynamic blocks or complex logic), much of the traditional PHP templating is replaced by block structures.
  • Impact for Developers:
    • Learning to think in blocks for entire site layouts.
    • Mastering theme.json for configuration and styling.
    • Developing custom blocks or block patterns becomes more central.
    • Less reliance on the Customizer for many theme options (though it can still be used for non-FSE contexts or specific needs).
  • Maturity: FSE is continuously evolving. While powerful, some advanced classic theme functionalities might still require custom solutions or have different approaches in a block theme context.
