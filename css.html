Modern CSS Cheatsheet

Focusing on modern CSS concepts, best practices, and avoiding obsolete techniques. Designed to complement the HTML cheatsheet using the same style.css file.

Basics & Syntax

CSS (Cascading Style Sheets) describes how HTML elements are to be displayed. Rules consist of a selector and one or more declarations.


selector {
  property: value; /* Declaration 1 */
  another-property: another-value; /* Declaration 2 */
}
  • Selector: Targets the HTML element(s) you want to style.
  • Declaration: Contains a property and its value.
  • Property: The style attribute you want to change (e.g., color, font-size).
  • Value: The value for the property (e.g., blue, 16px).
  • Each declaration ends with a semicolon (;).
  • Declarations are contained within curly braces ({ }).

Linking CSS to HTML

Use the <link> tag in the <head> section:


<!DOCTYPE html>
<html lang="en">
<head>
    
    <meta charset="UTF-8"> 
    <title>Page Title</title> 
    <link rel="stylesheet" href="style.css">
</head>
<body>
    
</body>
</html>

Selectors

Targeting elements is fundamental in CSS.

Selector Example Description
Type p Selects all <p> elements.
Class .my-class Selects all elements with class="my-class".
ID #my-id Selects the element with id="my-id" (should be unique).
Universal * Selects all elements.
Attribute [type="text"]
[data-abc]		 Selects elements with a specific attribute or attribute/value pair.
Descendant div p Selects all <p> elements inside a <div>.
Child ul > li Selects all <li> elements that are direct children of a <ul>.
Adjacent Sibling h2 + p Selects the first <p> immediately following an <h2>.
General Sibling h2 ~ p Selects all <p> elements that are siblings of an <h2> and come after it.
Grouping h1, h2, p Selects all <h1>, <h2>, and <p> elements.

Pseudo-Classes (:...) & Pseudo-Elements (::...)

Selectors that target elements based on their state or position (pseudo-classes) or style specific parts of an element (pseudo-elements).

Type Example Description
Pseudo-class a:hover Selects a link when the user hovers over it. Other common: :focus, :active, :visited, :first-child, :last-child, :nth-child(n), :not(selector), :where(selector), :is(selector).
Pseudo-element p::first-line Selects the first line of a <p>. Other common: ::before, ::after, ::marker, ::selection. (Note: single colon :before/:after also works for historical reasons).

Units

Ways to specify lengths, sizes, and other values.

Unit Type Description
px Absolute Pixels. Fixed size, dependent on screen resolution.
% Relative Percentage relative to the parent element's corresponding property.
em Relative Relative to the font-size of the parent element. (Can compound)
rem Relative Relative to the font-size of the root (<html>) element. (Recommended for typography scaling)
vw Relative 1% of the viewport's width.
vh Relative 1% of the viewport's height.
vmin Relative 1% of the minimum of the viewport's width and height.
vmax Relative 1% of the maximum of the viewport's width and height.
fr Relative (Grid) A fractional unit used in CSS Grid to represent a fraction of the available space.
auto Flexible Often used in Flexbox/Grid to take up available space or size based on content.

The Box Model

Every HTML element is treated as a box. Understanding its components is crucial for layout.


+---------------------+
|       Margin        |
| +-----------------+ |
| |     Border      | |
| | +-------------+ | |
| | |   Padding   | | |
| | | +---------+ | | |
| | | | Content | | | |
| | | +---------+ | | |
| | |             | | |
| | +-------------+ | |
| |                 | |
| +-----------------+ |
|                     |
+---------------------+
  • Content: The actual content (text, image, etc.). Sized by width/height.
  • Padding: Space between the content and the border. Affected by padding-top, padding-right, padding-bottom, padding-left, or shorthand padding.
  • Border: A line between the padding and the margin. Affected by border-width, border-style, border-color, or shorthand border.
  • Margin: Space outside the border. Affected by margin-top, margin-right, margin-bottom, margin-left, or shorthand margin. Margins can collapse vertically.

box-sizing Property

Controls how width and height are calculated.

  • box-sizing: content-box; (Default): width/height apply only to the content area. Padding and border are added *outside* the specified dimensions.
  • box-sizing: border-box;: width/height apply to the content + padding + border. This is often preferred as it makes layout calculations more intuitive.

Common practice: Apply box-sizing: border-box; globally or to most elements using a reset or base style.


*, *::before, *::after {
  box-sizing: border-box;
}

Display Property

Controls the layout behavior of an element.

  • display: block;: Starts on a new line and takes up the full width available (e.g., <p>, <div>, <h1>). Respects width, height, padding, margin.
  • display: inline;: Does not start on a new line and only takes up as much width as necessary (e.g., <span>, <a>, <strong>). Ignores width, height, top/bottom padding/margin.
  • display: inline-block;: Combines characteristics. Flows inline but respects width, height, padding, margin (e.g., <img>, <input> by default).
  • display: none;: Hides the element completely (removes it from document flow).
  • display: flex;: Makes the element a flex container, enabling Flexbox layout for its direct children.
  • display: grid;: Makes the element a grid container, enabling Grid layout for its direct children.

Layout: Flexbox

A one-dimensional layout method for arranging items in rows or columns.

Flex Container Properties (Apply to the parent with display: flex;)

  • flex-direction: row | row-reverse | column | column-reverse;: Defines the direction of items.
  • justify-content: flex-start | flex-end | center | space-between | space-around | space-evenly;: Aligns items along the main axis.
  • align-items: flex-start | flex-end | center | baseline | stretch;: Aligns items along the cross axis.
  • flex-wrap: nowrap | wrap | wrap-reverse;: Controls wrapping if items overflow the container.
  • gap: value;: Sets the gap between rows and columns (shorthand for row-gap and column-gap).

Flex Item Properties (Apply to direct children of the flex container)

  • flex-grow: number;: Ability to grow if necessary (default 0).
  • flex-shrink: number;: Ability to shrink if necessary (default 1).
  • flex-basis: value | auto;: Default size of the item before space is distributed.
  • flex: grow shrink basis;: Shorthand for flex-grow, flex-shrink, and flex-basis. Common values: 1 1 auto, auto, none.
  • order: number;: Specifies the order of a flexible item relative to the rest (default 0).
  • align-self: auto | flex-start | flex-end | center | baseline | stretch;: Overrides the container's align-items for a single item.

Example: Centering a div within its parent


.container {
  display: flex;
  justify-content: center; /* Horizontally center */
  align-items: center; /* Vertically center */
  min-height: 100vh; /* Example height */
}

Layout: Grid

A two-dimensional layout method for arranging items into columns and rows.

Grid Container Properties (Apply to the parent with display: grid;)

  • grid-template-columns: value(s);: Defines the columns of the grid (e.g., 1fr 1fr auto, repeat(3, 100px)).
  • grid-template-rows: value(s);: Defines the rows of the grid.
  • gap: value;: Sets the gap between rows and columns (shorthand for row-gap and column-gap).
  • justify-items: start | end | center | stretch;: Aligns items within their cells along the inline (row) axis.
  • align-items: start | end | center | stretch;: Aligns items within their cells along the block (column) axis.
  • grid-template-areas: "area1 area2" "area3 .";: Defines grid areas by naming them in a visual structure.

Grid Item Properties (Apply to direct children of the grid container)

  • grid-column: start / end;: Specifies the column size and location (e.g., 1 / 3, span 2). Shorthand for grid-column-start and grid-column-end.
  • grid-row: start / end;: Specifies the row size and location. Shorthand for grid-row-start and grid-row-end.
  • grid-area: row-start / column-start / row-end / column-end;: Shorthand for location/sizing, or can use area names defined in grid-template-areas.
  • justify-self: start | end | center | stretch;: Overrides the container's justify-items for a single item.
  • align-self: start | end | center | stretch;: Overrides the container's align-items for a single item.

Example: A simple 2-column grid


.container {
  display: grid;
  grid-template-columns: 1fr 2fr; /* Left column takes 1 part, right takes 2 parts */
  gap: 20px; /* Space between columns and rows */
}

Typography

Styling text content.

  • font-family: "Font Name", generic;: Specifies the font face. Always include generic fallbacks (serif, sans-serif, monospace, cursive, fantasy).
  • font-size: value;: Specifies the size of the font (e.g., 16px, 1em, 1.5rem).
  • font-weight: normal | bold | 100 | ... | 900;: Specifies the weight (boldness) of the font.
  • font-style: normal | italic | oblique;: Specifies the font style.
  • line-height: value | number;: Sets the height of a line box. Use unitless numbers (e.g., 1.5) for better inheritance.
  • text-align: left | right | center | justify;: Aligns the text within its containing block.
  • text-decoration: none | underline | overline | line-through;: Adds decoration lines to text. Use none to remove default link underlines.
  • color: value;: Sets the color of the text.

Loading Web Fonts (e.g., Google Fonts, custom fonts)


@font-face {
  font-family: "My Custom Font";
  src: url("my-custom-font.woff2") format("woff2"),
       url("my-custom-font.woff") format("woff");
  font-weight: normal;
  font-style: normal;
}

body {
  font-family: "My Custom Font", sans-serif;
}

Color & Background

Styling element colors and backgrounds.

  • color: value;: Sets the foreground color (text color).
  • background-color: value;: Sets the background color of an element.
  • background-image: url("path/to/image.jpg") | linear-gradient(...);: Sets the background image.
  • background-repeat: repeat | no-repeat | repeat-x | repeat-y;: How the background image repeats.
  • background-position: value;: Positions the background image (e.g., center top, 50% 50%).
  • background-size: auto | cover | contain | value;: Sizes the background image. cover fills the area, contain ensures the whole image is visible.
  • background: shorthand;: Shorthand property for background styles. Order matters: color image repeat position / size attachment origin clip.

Color Values

  • #RRGGBB or #RGB (Hexadecimal)
  • rgb(R, G, B) or rgba(R, G, B, A) (Red, Green, Blue, Alpha/Opacity)
  • hsl(H, S, L) or hsla(H, S, L, A) (Hue, Saturation, Lightness, Alpha)
  • Named Colors (e.g., red, blue, transparent)
  • Current Color (currentColor): Uses the element's computed color value.

Lesser Known / Modern CSS Features

Leveraging newer capabilities for cleaner, more flexible code.

  • CSS Variables (Custom Properties): Define reusable values. 
    
:root { /* Define globally */
  --primary-color: #1a73e8;
  --spacing-md: 16px;
}

h1 {
  color: var(--primary-color);
}

.section {
  margin-bottom: var(--spacing-md);
}
  • calc() function: Perform calculations to determine CSS values.
    Example: width: calc(100% - 40px);
  • clamp(), min(), max() functions: For responsive values based on constraints.
    Example: font-size: clamp(1rem, 2.5vw, 2rem); (Keeps font size between 1rem and 2rem, scaling fluidly based on viewport width, within 2.5vw).
  • object-fit & object-position: Control how the content of a replaced element (like <img> or <video>) should be resized to fit its container.
    Example: object-fit: cover; (Similar to background-size: cover).
  • filter property: Apply graphical effects like blur, grayscale, sepia, etc.
    Example: filter: grayscale(100%);
  • @layer (Cascade Layers): Organizes styles to control the cascade more explicitly. (Useful for design systems, third-party CSS). 
    
@layer reset, base, components, utilities;

@layer base {
  body { margin: 0; }
}

@layer components {
  .button { padding: 10px 20px; }
} /* Components layer has higher priority than base layer */
  • Container Queries (@container): Style elements based on the size of their *container*, not just the viewport. (Still relatively new support). 
    
.card-container {
  container-type: inline-size;
}

@container (min-width: 400px) {
  .card {
    display: flex; /* Card layout changes if container is wide enough */
  }
}

Key Concepts: Specificity, Cascade, Inheritance

Understanding these rules is crucial for predicting which styles apply.

Specificity

A score given to a CSS selector based on its components. Higher specificity wins when multiple rules target the same element and property.

  • Inline styles (highest)
  • IDs (#my-id) - 100 points
  • Classes (.my-class), Attributes ([type="text"]), Pseudo-classes (:hover) - 10 points
  • Elements (p), Pseudo-elements (::before) - 1 point
  • Universal selector (*), Combinators (>, +, ~), Negation (:not()) - 0 points (but inner selector counts)
  • !important (overrides specificity - use with caution)

Example: #my-id .my-class p (100 + 10 + 1 = 111 points) vs. .another-class p.highlight (10 + 1 + 10 = 21 points)

The Cascade

When multiple conflicting rules apply to an element, the browser resolves conflicts based on (in order of priority):

  1. Origin: User agent styles < User styles < Author styles < Author !important < User !important < User agent !important (highest)
  2. Layer: Styles in later @layer statements override those in earlier layers. Unlayered styles have higher priority than layered styles.
  3. Specificity: The rule with the highest specificity wins.
  4. Order: If specificity is equal, the last declared rule wins.

Inheritance

Some CSS properties (like color, font-family, text-align) are inherited by child elements if not explicitly set on the child. Others (like border, margin, padding, width) are not.

  • Use inherit keyword to explicitly inherit a property value from the parent.
  • Use initial to set a property to its default value.
  • Use unset to set a property to its inherited value if it's an inherited property, and to its initial value if not.
  • Use revert to reset a property to the value established by the user agent stylesheet (or user stylesheet if any).

Responsive Design (@media)

Adapting layouts and styles based on viewport characteristics, typically width.


/* Mobile-first approach: Default styles for small screens */
body {
  font-size: 1rem;
}

/* Styles for viewports wider than 768px */
@media (min-width: 768px) {
  body {
    font-size: 1.1rem;
  }
  .container {
    width: 700px;
    margin: 0 auto;
  }
}

/* Styles for viewports wider than 1200px */
@media (min-width: 1200px) {
  .container {
    width: 1100px;
  }
}
  • @media rule applies styles conditionally.
  • min-width: Applies styles when the viewport is *at least* the specified width. (Common for mobile-first).
  • max-width: Applies styles when the viewport is *at most* the specified width. (Common for desktop-first).
  • Other media features: orientation, resolution, prefers-color-scheme, prefers-reduced-motion, etc.

Good Practices

  • Mobile-First Approach: Design and style for small screens first, then use @media (min-width: ...) to add styles for larger screens. This is often more efficient.
  • Use CSS Variables: For consistent colors, spacing, fonts, etc. Makes updates much easier.
  • Use Semantic HTML: CSS selectors work best when targeting meaningful HTML elements and classes, not presentationally named classes (like `.red-text`).
  • Keep Selectors Simple: Avoid deep nesting. Prefer class selectors over complex combinators or ID selectors for styling where possible.
  • Use rem for Font Sizes: Allows users to scale text globally in browser settings. Use em for padding/margin where you want spacing relative to the *current* element's font size.
  • Use box-sizing: border-box;: Makes width and height calculations more intuitive.
  • Organize and Comment: Use comments (/* ... */) to section your CSS. Organize rules logically (e.g., by component, by section, using methodology like BEM, utility classes, etc.).
  • Optimize Performance: Minify CSS, use efficient selectors, consider critical CSS for initial load, defer non-critical CSS.
  • Prioritize Accessibility: Ensure sufficient color contrast, provide focus states for interactive elements (:focus), consider `prefers-reduced-motion` for animations.
  • Use Flexbox and Grid for Layout: They are the modern, powerful tools for arranging content.
  • Validate Your CSS: Use tools like the W3C CSS Validator.

Bad Practices (To Avoid)

  • Using !important Heavily: Destroys the cascade, makes debugging and overrides extremely difficult. Should be used sparingly, if at all (e.g., in utility classes or for overriding third-party styles).
  • Overly Specific or Deeply Nested Selectors: Leads to fragile code that's hard to maintain and reuse. Increases specificity issues.
  • Using IDs for Styling: IDs have high specificity, making them hard to override. They should primarily be used for JavaScript hooks or fragment identifiers in URLs. Use classes for styling.
  • Using Inline Styles (<div style="...">): Hard to manage, override, and violates separation of concerns. Highest specificity, difficult to remove or change later.
  • Using Tables for Layout: Tables are for data, not visual arrangement. Use CSS layout methods.
  • Ignoring Accessibility (e.g., low contrast, no focus outlines): Creates unusable experiences for many users.
  • Using Fixed Pixel Values for Everything: Makes responsive design harder. Use relative units like rem, %, vw where appropriate.
  • Hardcoding Colors/Values Repeatedly: Makes redesigns and updates tedious. Use CSS Variables.
  • Writing Duplicate Code: If you find yourself writing the same property/value pairs multiple times, consider grouping selectors or using a utility class/variable.
  • Relying on Browser Defaults Too Much: Browsers vary. Use a CSS reset or normalize stylesheet to ensure consistency.
