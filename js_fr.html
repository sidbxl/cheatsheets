Centré sur les fonctionnalités ECMAScript modernes (ES6+), les API du navigateur et les bonnes pratiques. Pas de var hérité ou de techniques obsolètes, sauf à des fins de comparaison.

Variables & Types de Données

Déclarer des Variables (ES6+)

  • let : Variable à portée de bloc, peut être réassignée. 
    
let message = "Bonjour";
message = "Monde";
  • const : Variable à portée de bloc, doit être initialisée, ne peut pas être réassignée. (Les objets/tableaux déclarés avec const peuvent toujours voir leur contenu modifié). 
    
const PI = 3.14;
// PI = 3.14159; // Erreur !

const personne = { nom: "Alice" };
personne.nom = "Bob"; // Ceci est autorisé

  • Évitez d'utiliser var en raison de sa portée de fonction et de ses particularités de hissage (hoisting).

Types de Données Primitifs

  • String (Chaîne de caractères) : Séquence de caractères.
    let nom = "Alice";
    Gabarits de Chaînes (Template Literals ES6) : let salutation = `Bonjour, ${nom} !`;
  • Number (Nombre) : Entiers et nombres à virgule flottante.
    let age = 30; let prix = 19.99;
  • Boolean (Booléen) : true ou false.
    let estActif = true;
  • Null : Représente l'absence intentionnelle de toute valeur d'objet.
    let donnees = null;
  • Undefined (Indéfini) : Une variable qui a été déclarée mais à laquelle aucune valeur n'a été assignée.
    let score; // score est undefined
  • Symbol (ES6) : Valeur primitive unique et immuable, souvent utilisée comme clé d'objet.
    const id = Symbol('id');
  • BigInt (ES2020) : Pour les entiers plus grands que Number.MAX_SAFE_INTEGER.
    const grandNombre = 1234567890123456789012345678901234567890n;

Types de Données Non Primitifs (Objets)

  • Object (Objet) : Collection de paires clé-valeur. 
    
let utilisateur = {
  prenom: "John",
  nomDeFamille: "Doe",
  age: 25
};
  • Array (Tableau) : Liste ordonnée de valeurs. 
    
let couleurs = ["rouge", "vert", "bleu"];
  • Function (Fonction) : Un bloc de code conçu pour effectuer une tâche particulière. (Les fonctions sont aussi des objets).

Fonctions

  • Déclaration de Fonction : 
    
function saluer(nom) {
  return `Bonjour, ${nom} !`;
}
  • Expression de Fonction : 
    
const saluer = function(nom) {
  return `Bonjour, ${nom} !`;
};
  • Fonctions Fléchées (ES6) : Syntaxe concise, liaison lexicale de this. 
    
const ajouter = (a, b) => a + b; // Retour implicite pour une seule expression

const multiplier = (a, b) => {
  return a * b; // Retour explicite pour multi-lignes
};
  • Paramètres :
    • Paramètres par Défaut (ES6) : function direSalut(nom = 'Invité') { /* ... */ }
    • Paramètres du Reste (Rest Parameters ES6) : Collecte les arguments restants dans un tableau. 
      
function somme(...nombres) {
  return nombres.reduce((acc, actuel) => acc + actuel, 0);
}
somme(1, 2, 3); // 6
  • Portée (Scope) : Les variables déclarées avec let et const ont une portée de bloc. Les fonctions créent leur propre portée.

Tableaux (Arrays)

  • Tableaux Littéraux : const fruits = ['pomme', 'banane', 'cerise'];
  • Accéder aux Éléments : Index basé sur zéro : fruits[0] // 'pomme'
  • Propriétés & Méthodes Courantes :
    • .length : Nombre d'éléments.
    • .push(element) : Ajoute à la fin. .pop() : Supprime de la fin.
    • .unshift(element) : Ajoute au début. .shift() : Supprime du début.
    • .indexOf(element), .includes(element) (ES7)
    • .slice(debut, fin) : Retourne un nouveau tableau (non-mutant).
    • .splice(debut, nbASupprimer, ...elementsAAjouter) : Modifie le tableau original.
  • Méthodes Modernes d'Itération/Transformation (ES6+) : (Celles-ci ne modifient pas le tableau original et retournent un nouveau tableau ou une valeur) 
    
const nombres = [1, 2, 3, 4, 5];

nombres.forEach(nb => console.log(nb * 2));

const doubles = nombres.map(nb => nb * 2); // [2, 4, 6, 8, 10]

const pairs = nombres.filter(nb => nb % 2 === 0); // [2, 4]

const somme = nombres.reduce((accumulateur, valeurActuelle) => accumulateur + valeurActuelle, 0); // 15

const premierPair = nombres.find(nb => nb % 2 === 0); // 2

const aUnPair = nombres.some(nb => nb % 2 === 0); // true

const tousPositifs = nombres.every(nb => nb > 0); // true
  • Syntaxe de Décomposition (Spread ES6) : Pour copier ou fusionner des tableaux. 
    
const tab1 = [1, 2];
const tab2 = [...tab1, 3, 4]; // [1, 2, 3, 4]
const copie = [...tab1];
  • Déstructuration de Tableau (ES6) : 
    
const [premier, deuxieme, ...reste] = ['pomme', 'banane', 'cerise', 'datte'];
console.log(premier); // 'pomme'

JavaScript Asynchrone

Gérer les opérations qui prennent du temps (par ex., requêtes réseau, minuteurs) sans bloquer le thread principal.

Callbacks (Traditionnel)

Fonctions passées en arguments pour être exécutées ultérieurement. Peut mener à "l'enfer des callbacks" (callback hell) si imbriquées profondément.

Promesses (Promises ES6)

Un objet représentant l'achèvement (ou l'échec) éventuel d'une opération asynchrone.


const maPromesse = new Promise((resolve, reject) => {
  // Opération asynchrone
  const succes = true; // ou false
  if (succes) {
    resolve('Opération réussie !');
  } else {
    reject('Opération échouée.');
  }
});

maPromesse
  .then(resultat => console.log(resultat)) // Gérer le succès
  .catch(erreur => console.error(erreur)) // Gérer l'échec
  .finally(() => console.log('Opération terminée.')); // S'exécute quel que soit le succès/échec (ES2018)

// Promise.all([promesse1, promesse2]), Promise.race([promesse1, promesse2])

Async/Await (ES2017)

Sucre syntaxique par-dessus les Promesses, rendant le code asynchrone plus semblable à du code synchrone.


async function recupererDonnees() {
  try {
    const reponse = await fetch('https://api.example.com/data');
    if (!reponse.ok) {
      throw new Error(`Erreur HTTP ! statut : ${reponse.status}`);
    }
    const donnees = await reponse.json();
    console.log(donnees);
  } catch (erreur) {
    console.error('Impossible de récupérer les données :', erreur);
  }
}
recupererDonnees();

Modules (Modules ES6)

Organiser le code en morceaux réutilisables. Doit être servi depuis un serveur (pas file://) et utilisé avec <script type="module">.

math.js (Exemple de Module) :


export const PI = 3.14159;
export function ajouter(a, b) {
  return a + b;
}
export default function multiplier(a, b) { // Un seul export par défaut par module

}

Gestion des Erreurs


try {
  // Code susceptible de lever une erreur
  let resultat = operationRisquee();
  console.log(resultat);
} catch (erreur) {
  // Code pour gérer l'erreur
  console.error('Une erreur est survenue :', erreur.message);
  // erreur.name, erreur.stack
} finally {
  // Code qui s'exécute quel que soit l'erreur (optionnel)
  console.log('Opération terminée.');
}

// Lever des erreurs personnalisées
function diviser(a, b) {
  if (b === 0) {
    throw new Error('La division par zéro n\'est pas autorisée.');
  }
  return a / b;
}

Stockage Navigateur (Environnement Navigateur)

Stocker des données dans le navigateur de l'utilisateur.

localStorage

Stocke les données sans date d'expiration. Les données persistent même après la fermeture de la fenêtre du navigateur.


// Définir un élément
localStorage.setItem('nomUtilisateur', 'Alice');

// Obtenir un élément
const nomUtilisateur = localStorage.getItem('nomUtilisateur'); // 'Alice'

// Supprimer un élément
localStorage.removeItem('nomUtilisateur');

// Effacer tous les éléments
// localStorage.clear();

// Note : localStorage ne stocke que des chaînes. Utilisez JSON.stringify/parse pour les objets.
const utilisateur = { nom: 'Bob', id: 123 };
localStorage.setItem('utilisateur', JSON.stringify(utilisateur));
const utilisateurRecupere = JSON.parse(localStorage.getItem('utilisateur'));

sessionStorage

Stocke les données pour une session (les données sont perdues lorsque l'onglet du navigateur est fermé). L'API est la même que localStorage.


sessionStorage.setItem('donneesSession', 'Info temporaire');

JSON (JavaScript Object Notation)

Un format d'échange de données léger. Facile à lire/écrire pour les humains, facile à analyser/générer pour les machines.

  • JSON.stringify(objet) : Convertit un objet ou une valeur JavaScript en une chaîne JSON. 
    
const donnees = { nom: "Test", valeur: 42 };
const chaineJson = JSON.stringify(donnees);
// chaineJson est "{\"nom\":\"Test\",\"valeur\":42}"
  • JSON.parse(chaineJson) : Analyse une chaîne JSON, construisant la valeur ou l'objet JavaScript décrit par la chaîne. 
    
const objetJson = JSON.parse(chaineJson);
// objetJson est { nom: "Test", valeur: 42 }

Bonnes Pratiques

  • Utiliser le Mode Strict : 'use strict'; au début de vos scripts ou fonctions aide à détecter les erreurs de codage courantes. (Les modules sont en mode strict par défaut).
  • Utiliser let et const : Préférez const par défaut ; utilisez let si une réassignation est nécessaire. Évitez var.
  • Écrire du Code Clair et Lisible : Utilisez des noms de variables/fonctions significatifs, gardez les fonctions petites et ciblées, commentez la logique complexe.
  • Adopter les Fonctionnalités Modernes : Fonctions fléchées, gabarits de chaînes, déstructuration, spread/rest, Promesses, async/await rendent le code plus concis et lisible.
  • Gérer les Erreurs avec Élégance : Utilisez try...catch pour les erreurs synchrones et .catch() ou try...catch avec async/await pour les erreurs asynchrones.
  • Éviter les Variables Globales : Encapsulez le code dans des fonctions ou des modules pour éviter de polluer la portée globale.
  • Utiliser l'Égalité Stricte : Préférez === et !== à == et != pour éviter les surprises de coercition de type.
  • Optimiser les Manipulations du DOM : Minimisez l'accès direct au DOM. Regroupez les mises à jour ou utilisez des fragments de document si vous créez de nombreux éléments.
  • Utiliser la Délégation d'Événements : Pour de meilleures performances lors de la gestion d'événements sur plusieurs éléments enfants.
  • Garder le Code DRY (Ne Vous Répétez Pas) : Créez des fonctions ou des modules réutilisables pour les tâches courantes.
  • Linting & Formatage du Code : Utilisez des outils comme ESLint et Prettier pour maintenir la qualité et la cohérence du code.

Mauvaises Pratiques (Choses à Éviter)

  • Utiliser var : Entraîne des problèmes de hissage (hoisting) et une portée moins claire.
  • Modifier les Prototypes Intégrés (par ex., Array.prototype.maMethode) : Peut entraîner des conflits avec les futures versions de JS ou d'autres bibliothèques.
  • Utiliser eval() : Risque de sécurité et impact sur les performances. Il y a presque toujours une meilleure façon de faire.
  • Utiliser document.write() après le chargement de la page : Peut écraser toute la page.
  • Bloquer le Thread Principal : Les opérations synchrones de longue durée gèleront l'interface utilisateur. Utilisez des modèles asynchrones.
  • Créer un "Enfer de Callbacks" : Les callbacks profondément imbriqués sont difficiles à lire et à maintenir. Utilisez les Promesses ou async/await.
  • Ignorer la Gestion des Erreurs : Peut entraîner des plantages ou des comportements inattendus.
  • Utiliser innerHTML avec des Entrées Utilisateur Non Validées : Risque de sécurité (attaques XSS). Utilisez textContent pour le texte ou validez soigneusement le HTML.
  • Se Fier Implicitement à la Coercition de Type : Peut entraîner des bogues. Soyez explicite sur les types ou utilisez des comparaisons strictes.
  • Écrire du code trop complexe ou "intelligent" : Donnez la priorité à la lisibilité et à la maintenabilité.
