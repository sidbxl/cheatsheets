Modern HTML Cheatsheet

Focusing on HTML as a living standard, semantic markup, and accessibility. No obsolete or deprecated tags.

Minimal Tags for an HTML Page

The absolute minimum required for a valid HTML document:


<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
</head>
<body>
    
</body>
</html>
  • <!DOCTYPE html>: Declares the document type (essential).
  • <html lang="en">: The root element. lang attribute is important for accessibility and search engines.
  • <head>: Contains meta-information about the HTML document (not displayed).
    • <meta charset="UTF-8">: Specifies character encoding (UTF-8 is highly recommended).
    • <meta name="viewport" content="width=device-width, initial-scale=1.0">: Configures the viewport for responsive web design.
    • <title>Page Title</title>: The title that appears in the browser tab or window title bar. Crucial for SEO and accessibility.
  • <body>: Contains the visible page content.

Rules for Escaping Characters (HTML Entities)

To display characters that have special meaning in HTML, or characters not easily typed, use HTML entities:

Character Named Entity Numeric Entity Description
< &lt; &#60; Less than sign
> &gt; &#62; Greater than sign
& &amp; &#38; Ampersand
" &quot; &#34; Double quote (Needed in attribute values enclosed by double quotes)
' &apos; &#39; Single quote (Needed in attribute values enclosed by single quotes)
  &nbsp; &#160; Non-breaking space
© &copy; &#169; Copyright symbol

Example: To display <p> as text, write &lt;p&gt;.

Must-Know HTML Tags

Document Structure & Semantics

  • <header>: Introductory content for a section or page (logos, navigation, search).
  • <nav>: A section of navigation links.
  • <main>: The main content of the document. There should only be one <main> element per page that is not hidden.
  • <article>: Self-contained, independently distributable content (e.g., blog post, news story).
  • <section>: A thematic grouping of content, typically with a heading.
  • <aside>: Content tangentially related to the content around it (e.g., sidebar, pull quotes).
  • <footer>: Footer for a section or page (copyright, related links).
  • <h1> to <h6>: Heading levels. Use them hierarchically.
  • <p>: A paragraph of text.
  • <div>: A generic container for flow content. Use when no other semantic element is appropriate (often for styling/layout purposes with CSS).
  • <span>: A generic inline container for phrasing content. Use for styling parts of text or when no other semantic inline element is appropriate.

Text Content

  • <a href="url">Link Text</a>: A hyperlink.
    • target="_blank": Opens link in a new tab/window. Add rel="noopener noreferrer" for security.
    • download: Prompts user to download the linked resource.
  • <strong>: Indicates text with strong importance, seriousness, or urgency. (Typically bold).
  • <em>: Indicates stress emphasis. (Typically italic).
  • <b>: Stylistically offset text without conveying extra importance (e.g., keywords). Use CSS for styling if possible.
  • <i>: Text in an alternate voice or mood (e.g., technical term, foreign phrase). Use CSS for styling if possible.
  • <br>: A line break. Use sparingly, only for actual line breaks within text (like poems or addresses).
  • <hr>: A thematic break (e.g., scene change in a story, shift of topic). Often displayed as a horizontal rule.
  • <blockquote>: Represents a section quoted from another source. Use cite attribute for the source URL.
  • <q>: An inline quotation. Browsers often add quotation marks. Use cite attribute for source URL.
  • <cite>: Represents a reference to a creative work (e.g., book title, song, film).
  • <sub> & <sup>: Subscript and superscript text.
    Example: H2O, E = mc2 (Live HTML example)
  • <ins> & <del>: Represents inserted or deleted text, often for edits.
    Example: This text was removed added. (Live HTML example)

Lists

  • <ul>: An unordered list (bullet points).
  • <ol start="number" reversed type="a/A/i/I/1">: An ordered list (numbered).
    • start: Starting number.
    • reversed: Reverses numbering.
    • type: Numbering style (a, A, i, I, 1).
  • <li>: A list item (used in <ul>, <ol>, <menu>).
  • <dl>, <dt>, <dd>: Description list, term, and description. 
    
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
</dl>

Images & Media

  • <img src="path/to/image.jpg" alt="Descriptive text" width="X" height="Y">: Embeds an image.
    • src: Path to the image.
    • alt: Essential for accessibility (alternative text if image doesn't load or for screen readers). Leave empty (alt="") for purely decorative images.
    • width & height: Specify dimensions to prevent layout shifts. Can be overridden by CSS. Use intrinsic dimensions if possible.
    • srcset & sizes: For responsive images (providing multiple image sources for different resolutions/viewport sizes).
  • <audio controls src="audio.mp3">Fallback text</audio>: Embeds audio content.
    Can use <source src="audio.ogg" type="audio/ogg"> for multiple formats.
  • <video controls width="X" height="Y" src="video.mp4" poster="image.jpg">Fallback text</video>: Embeds video content.
    Can use <source src="video.webm" type="video/webm"> for multiple formats.
  • <iframe src="url" title="Descriptive title"></iframe>: Embeds another HTML document. title attribute is important for accessibility.

Forms

Crucial for user interaction. Always associate labels with form controls.


<form action="/submit-url" method="post">
    <label for="username">Username:</label>
    <input type="text" id="username" name="username" required>

    <input type="submit" value="Submit">
</form>
  • <form action="url" method="get|post">: Defines an HTML form.
  • <label for="elementId">Label text</label>: Associates a label with a form control. Clicking the label focuses/activates the control.
  • <input type="...">: Most versatile form element.
    • Common type attributes: text, password, checkbox, radio, submit, button, file, date, email, number, tel, url, search, color, range, hidden.
    • Common attributes: id, name, value, placeholder, required, disabled, readonly, checked (for radio/checkbox), min, max, step (for number/range).
  • <textarea name="message" rows="4" cols="50"></textarea>: A multi-line text input.
  • <select name="options"> & <option value="val">Text</option>: A dropdown list.
    • <optgroup label="Group Name">: Groups options within a <select>.
  • <button type="submit|reset|button">Button Text</button>: A clickable button. Can contain HTML content, unlike <input type="button">.
  • <fieldset> & <legend>: Groups related controls within a form, with a caption. 
    
<fieldset>
  <legend>Contact Details</legend>
  
</fieldset>
  • <datalist id="items"> & <option value="Choice">: Provides pre-defined options for an <input> element (autocomplete). Link with <input list="items">.

  • <table>: The main table container.
  • <caption>: Table title/caption. Important for accessibility.
  • <thead>: Groups header content in a table.
  • <tbody>: Groups body content in a table.
  • <tfoot>: Groups footer content in a table.
  • <tr>: A table row.
  • <th scope="col|row">: A table header cell. scope attribute improves accessibility.
  • <td>: A table data cell.
  • colspan / rowspan attributes on <th> or <td>: Make a cell span multiple columns/rows.

Good Practices

  • Semantic HTML: Use HTML tags for their intended purpose. This improves accessibility, SEO, and maintainability. (e.g., use <button> for buttons, not a styled <div>).
  • Accessibility (a11y):
    • Always use alt text for images (alt="" for decorative images).
    • Ensure sufficient color contrast.
    • Use ARIA (Accessible Rich Internet Applications) attributes when necessary to provide additional semantics for assistive technologies, but prioritize native HTML semantics.
    • Ensure keyboard navigability.
    • Use <label> for form controls.
  • Validate Your HTML: Use tools like the W3C Validator to catch errors.
  • Separate Concerns:
    • HTML for structure.
    • CSS for presentation/styling.
    • JavaScript for behavior/interactivity.
    Avoid inline styles (<div style="...">) and inline script event handlers (<button onclick="...">) where possible.
  • Proper Indentation & Readability: Keep your code clean and well-formatted.
  • Comments: Use comments (<!-- ... -->) to explain complex parts of your markup.
  • Use Headings Hierarchically: Don't skip heading levels (e.g., <h1> then <h3>). Structure your content logically.
  • Character Encoding: Always declare <meta charset="UTF-8">.
  • Language Declaration: Set the language of your page with <html lang="en">.
  • Responsive Design: Include <meta name="viewport" ...> and use responsive techniques (CSS media queries, flexible grids).
  • Close Tags: While some tags are self-closing (e.g. <img>, <br>, <input>) or optional in some contexts, it's generally good practice to close all non-void elements (e.g. <p>...</p>, <div>...</div>). In XHTML-syntax (like in JSX), self-closing tags require a trailing slash (<img />). HTML syntax for void elements does not.

Bad Practices (To Avoid)

  • Using Tables for Layout: Tables are for tabular data. Use CSS (Flexbox, Grid) for page layout.
  • Using Inline Styles Extensively: <p style="color: red;"> makes maintenance hard. Use external CSS files.
  • Not Using alt Text for Images: This is bad for accessibility and SEO.
  • Using Deprecated Tags/Attributes: E.g., <font>, <center>, <bgcolor>, <frame>, <frameset>, <marquee>. They are no longer part of the HTML standard.
  • "Divitis" or "Spanitis": Overusing <div> and <span> when more semantic tags are appropriate.
  • Skipping Heading Levels: E.g., going from an <h1> directly to an <h3>. This breaks document structure and confuses assistive technologies.
  • Using <br> for Spacing: Use CSS margins/paddings for layout spacing.
  • Not Closing Tags Properly: While browsers are forgiving, it can lead to unexpected rendering issues and is invalid for some elements.
  • Putting Block-Level Elements Inside Inline Elements: For example, <a href="#"><div>...</div></a> is generally incorrect (though HTML5 parsing rules are more lenient for <a>, it's still better to avoid if a <div> can just be styled as a link, or the link wraps inline content). The rule is: phrasing content (inline) can go inside flow content (block), but generally not the other way around, except for specific elements like <a> which can wrap flow content.
  • Writing Generic Link Text: "Click here" or "Read more" is bad for accessibility. Link text should be descriptive of the link's destination.
